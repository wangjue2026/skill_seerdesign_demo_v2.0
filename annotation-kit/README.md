# 交互自助标注工具 (Annotation Kit - 100% 无依赖原生 JS 版)

## 目的

本工具套件提供一个 **全屏自适应、高级暗黑风格** 的页面标注方案，让您的 HTML 项目拥有一套极具科技感（Live Dev Agent 风格）的批注控制台。

**【全新架构升级】**：已从 Vue 3 彻底重构为 **Vanilla JS (原生 Web Component)**。不依赖任何框架，内联嵌入目标 HTML，实现真正的**单文件离线便携**。

---

## 核心特性

1. **100% 零依赖与跨平台**：只需原生浏览器，不造成白屏、请求阻断或框架冲突。
2. **单文件离线便携**：组件 CSS 与 JS 全部内联到目标 `.html` 中，"一个 HTML 走天下"。
3. **Shadow DOM 样式隔离**：右侧悬浮面板绝不挤压或污染宿主页面的业务样式。
4. **百分比坐标打点**：响应式百分比坐标，跨设备、跨分辨率精确定位。
5. **文件内数据持久化**：打点数据以 JSON 形式内嵌在 HTML 文件中，随文件一起流转。
6. **一键复制 Markdown**：「交互说明」面板右上角的「📝 复制 MD」按钮，可将所有批注一键生成结构化 Markdown 文档并复制到剪贴板，方便粘贴至需求文档、飞书文档等协作平台。
7. **支持截图粘贴与图片上传**：支持在打标弹窗中通过 `Ctrl+V / Cmd+V` 粘贴屏幕截图，或点击选择本地图片上传，图片以 Base64 格式保存，且支持在 Markdown 中同步复制导出。

## 使用方式

### 第一步：注入面板

在终端执行以下命令，将打标工具注入到目标 HTML 文件：

```bash
node annotation-kit/annotation-tool.js demo.html
```

执行完毕后，`demo.html` 底部会自动插入：
- 工具代码块（`<!-- SASE Annotation Kit Start -->` 到 `<!-- SASE Annotation Kit End -->`）
- 数据承载块（`<script id="sase-annotations" type="application/json">...</script>`）

> **重要**：数据块必须位于 `</body>` 标签之前，这是浏览器能够读取到它的前提条件。注入脚本会自动保证这一点。

---

### 第二步：进行标注与打点

直接在浏览器中打开注入后的 HTML（支持 `file:///` 协议，无需本地服务器）：

- **快捷打点**：点击右下角胶囊的 `📍 打点` 按钮，在屏幕任意位置点击，输入标题和内容。
- **粘贴/上传截图**：在弹出的打标编辑窗口中，可在任意位置按 `Ctrl+V` (或 `Cmd+V`) 粘贴系统剪贴板中的截图，或者点击虚线图片区域来上传本地图片。点击「✕ 删除图片」可以随时移除。
- **展开控制台**：点击 `🛠️ 控制台` 可滑出右侧的完整批注面板。
- **显示/隐藏点位**：胶囊上的 `说明` 开关可一键隐藏画布上的打点圆点。
- **复制 Markdown**：在「📋 交互说明」标签页，标题行右侧有「📝 复制 MD」按钮，点击后将所有批注格式化为 Markdown 并复制到剪贴板（可直接粘贴到飞书、Notion、语雀等文档协作平台，截图也会以 Base64 图片格式内嵌渲染）。

---

### 第三步：保存数据并分享（⚠️ 重要！请仔细阅读）

这是最容易出问题的步骤，请严格按照以下流程操作：

#### 正确流程

1. 打标完成后，点击右下角胶囊 → `🛠️ 控制台`
2. 切换到 **「💻 源码数据」** 标签页
3. 点击绿色按钮 **「📋 复制最新源码」**
4. 打开 **HTML 文件** （用任意文本编辑器，如 VS Code、记事本）
5. 滚动到文件**最底部**，找到如下结构的代码块：

   ```html
   <script id="sase-annotations" type="application/json">
   { "timestamp": ..., "comments": [...] }
   </script>
   ```

6. **将整个区块（从 `<script` 到 `</script>`）完整替换**为刚才复制的内容
7. 保存文件，即可分享

---

#### ❌ 常见错误操作及后果

| 错误操作 | 后果 |
|---------|------|
| 复制数据后直接粘贴到文件**末尾**（在 `</html>` 之后） | 文件中存在多个数据块，且新的块在 `</html>` 之外，浏览器读取不到，接收方看不到标注 |
| 粘贴时没有删除旧数据块，两个块同时存在 | 工具虽然能智能选最新的一个，但旧块冗余在文件里容易造成混乱 |
| 复制后粘贴到 `</body>` 之后、`</html>` 之前 | 部分浏览器可能无法读取，不推荐 |

---

#### ✅ 正确粘贴位置示意

```html
  <!-- ... 页面内容 ... -->

  <script id="sase-annotations" type="application/json">
  {
    "timestamp": 1779785187979,
    "comments": [...]  ← 这里是您的标注数据
  }
  </script>
</body>           ← 数据块必须在这行之前！
</html>
```

## 常见问题

| 问题 | 解答 |
|------|------|
| 发给别人后标注不见了？ | 最常见原因：复制的数据块粘贴在了 `</html>` 之后，浏览器读取不到。请按上方「正确粘贴位置示意」检查。 |
| 刷新页面后我刚打的点不见了？ | 打点后数据暂存在浏览器 `localStorage` 中，必须完成「复制→替换→保存」才能真正持久化到文件里。 |
| 接收方能修改标注吗？ | 可以，他们同样可以在浏览器中进行打点操作，再通过同样的「复制最新源码」流程保存。 |
| 如何撤销注入、还原干净的 HTML？ | 删除 `<!-- SASE Annotation Kit Start -->` 到 `<!-- SASE Annotation Kit End -->` 之间的代码块，以及底部的 `<script id="sase-annotations">` 数据块即可。 |
| 注入脚本会修改原始页面样式吗？ | 不会，工具采用 Shadow DOM 完全隔离，绝不影响宿主页面的任何现有样式。 |
