### 分组标题规范

分组标题用于菜单分组，只起到视觉分割作用，不是具体的菜单项。

| 类型 | 字号 | 字重 | 颜色 | 左边距 | 高度 | 选中态 | 交互 |
|------|------|------|------|--------|------|--------|------|
| 一级分组标题 | 12px | 400 | var(--color-graphite) (#A1A7B3) | 16px | 32px | 无 | 不可点击 |
| 二级分组标题 | 12px | 400 | var(--color-graphite-d20) (#5E6573) | 40px | 32px | 无 | 不可点击 |

**核心特性**：
- 不可点击，仅起视觉分隔作用
- 没有选中态，永远不会被标记为选中
- 不影响其他菜单的层级结构
- 不参与菜单的展开/收起逻辑
- 不参与菜单的历史记录保存与恢复
- 收起态下分组标题自动隐藏
- 悬浮态/焦点态/激活态均透明，无任何背景或颜色变化

## 4. 视觉状态红线 (Visual States)

### 4.1 选中态 (Selected)

| 属性 | 规范值 |
|-----|--------|
| 背景 | 蓝色渐变，从左侧 rgba(28,110,255,0.2) 渐变至右侧透明 |
| 左侧激活条 | 3px 垂直蓝色条，紧贴左边缘，z-index: 1 |
| 文字颜色 | 纯白 #FFFFFF |
| 图标颜色 | 激活色 #4BFFC6 (带 3px 蓝色投影 `drop-shadow(0 0 3px rgba(75,255,198,0.2))`) |
| 图标位置 | 不变（选中与未选中图标和文字位置完全一致） |
| 一级菜单字重 | 600（未选中为 400） |
| 二级菜单字重 | 600（未选中为 400） |

**适用范围**：
- 二级菜单可进入选中态
- 一级菜单如果没有包含子菜单（即本身为叶子节点），也可以进入选中态，此时图标变蓝（#1C6EFF）
- 一级菜单如果包含子菜单，则不进入选中态，而是触发父级高亮（见 4.2）
- 分组标题不可进入选中态

**重要约束**：
- 选中的蓝色条不能影响菜单图标和文字的位置
- 选中和未选中时候图标和文字的位置是不变的，不要产生右移

### 4.2 父级高亮 (Parent Active)

当一级菜单包含二级菜单且子菜单选中时，触发以下效果：

| 属性 | 规范值 |
|-----|--------|
| 一级菜单图标颜色 | #1C6EFF (blue) |
| 一级菜单文字颜色 | 保持原色（白色），不变色 |
| 一级菜单底色 | #1E232B |
| 同层级二级菜单底色 | #1E232B |
| 同层级二级分组标题底色 | #1E232B |

**重要约束**：
- 二级菜单被选中时，该二级菜单所在的一级菜单以及同一层级的二级菜单的底色改为 #1E232B
- 一级菜单仅图标变色（blue），文字保持白色不变

## 5. 交互与联动逻辑

### 5.1 自动展开与互斥收起

| 规则 | 说明 |
|------|------|
| 默认展开 | 选中二级菜单时，该二级菜单所在的一级菜单必须默认展开，不能收起 |
| 保证可见 | 确保选中的菜单能够直接被看到 |
| 互斥收起 | 选中二级菜单时，仅保留该二级菜单所属的一级菜单展开，自动收起其他已展开的一级菜单；点击一级菜单展开/收起时不影响其他已展开的一级菜单 |

**实现逻辑**：
1. 维护当前展开的一级菜单 Key 列表
2. 点击一级菜单时，仅切换该菜单的展开/收起状态，不影响其他已展开的一级菜单
3. 选中二级菜单时，仅保留该二级菜单所属的一级菜单展开，收起其他所有一级菜单
4. 排除小标题：小标题不参与展开/收起逻辑，因为没有子菜单

### 5.2 折叠交互

| 规则 | 说明 |
|------|------|
| 收起态宽度 | 54px |
| Hover 交互 | 鼠标 Hover 区域需临时悬浮展开完整菜单，浮层带阴影效果 |
| 恢复机制 | 移开后恢复 54px 收起态 |
| 收起态隐藏 | 文字、产品名称、用户名、展开箭头、分组标题全部隐藏 |
| 收起态图标 | 一级菜单图标水平居中，左边距 19px（(54-16)/2） |
| 收起态用户区 | 头像居中显示，取消左间距 |

## 6. 样式强制装配要点

以下为必须强制实现的视觉规范，无论使用何种技术栈均需确保：

1. **一级菜单图标位置**：左边距 16px，尺寸 16px × 16px，与文字间距 8px
2. **二级菜单圆点位置**：左边距 40px，尺寸 4px × 4px，与文字间距 8px
3. **一级分组标题左边距**：16px，不可点击，无选中态
4. **二级分组标题左边距**：40px，不可点击，无选中态
5. **选中态蓝色激活条**：3px 垂直蓝色边框 + 蓝色渐变底色（从左到右，20%蓝色透明度到透明），必须不影响图标和文字的原始位置
6. **父级高亮**：子菜单选中时，一级菜单图标变蓝（#1C6EFF），文字不变色，同层级底色 #1E232B
7. **叶子节点选中**：无子菜单的一级菜单选中时，图标变蓝（#1C6EFF）
8. **一级菜单字号**：13px，选中字重 600，未选中字重 400
9. **二级菜单字号**：12px，选中字重 600，未选中字重 400
10. **分组标题颜色**：一级分组标题 var(--color-graphite) (#A1A7B3)，二级分组标题 var(--color-graphite-d20) (#5E6573)，悬浮/焦点/激活态均透明
11. **互斥展开**：选中二级菜单时仅保留其所属一级菜单展开，点击一级菜单时不影响其他已展开的一级菜单
12. **收起态**：54px 宽度，图标居中，文字和分组标题隐藏

## 8. 设计原则

| 原则 | 说明 |
|------|------|
| 深色一致性 | 侧边栏整体保持 #272E38 深色背景，所有状态变化在深色基础上叠加 |
| 坐标严格性 | 一级菜单图标 16px 左边距、二级菜单 40px 起始坐标必须严格执行 |
| 布局稳定性 | 选中态蓝色条不影响图标和文字的原始位置 |
| 互斥展开 | 选中二级菜单时仅保留其所属一级菜单展开，点击一级菜单展开/收起不影响其他 |
| 父级联动 | 子菜单选中时父级图标变蓝 + 同层级底色变化，形成视觉联动 |
| 分组纯视觉 | 一级和二级分组标题仅起视觉分割作用，不参与任何交互逻辑 |
