# aTrust 零信任标准导航规范

## 使用声明

本规范只定义 aTrust 导航的视觉、结构、交互和数据规则，不提供任何具体代码实现。

生成代码时，必须以当前项目的技术栈 Skill 为准。

如果本规范中的设计描述与项目技术栈约束发生冲突，优先遵守项目技术栈约束。

本规范禁止包含：
- React / Vue / Angular 代码示例
- JSX / TSX / TypeScript 示例
- import 语句
- 具体组件库 API 示例
- 具体图标组件名称
- 具体框架状态管理代码
- 具体 Ant Design / Ant Design Vue 内部 class 选择器

本规范中的组件名称如 Header、Sider、Tabs、Select、Menu 仅表示设计语义，不代表必须使用某个框架或组件库的同名组件。

## 2. 顶部导航栏 Header 视觉规范

### 2.1 布局权重重组 (Layout Weight)

Header 采用三栏 Flex 布局，确保一级菜单绝对居中。

| 区域 | 内容 | Flex 设置 | 说明 |
|-----|------|----------|------|
| 左侧 (Brand Area) | Logo、产品名称、订阅切换器 | `flex: 0 0 auto` | 宽度由内容撑开 |
| 中间 (NavCenter) | 一级菜单 Tabs | `flex: 1` + `justify-content: center` | 在剩余空间内绝对居中 |
| 右侧 (Actions Area) | 搜索图标、登录账号 | `flex: 0 0 auto` | 宽度由内容撑开 |

### 3.3 文本对齐红线 (The 48px Reading Line)

为了确保侧边栏在视觉上维持一条绝对整齐的阅读垂直线，必须强制对齐各级菜单的文字起始位置。

#### 二级菜单 (Level 2)
| 属性 | 规范值 |
|-----|--------|
| 图标位置 | 左边距固定 24px |
| 图标尺寸 | 16px × 16px |
| 图标与文字间距 | 8px |
| **文字起始坐标** | **48px** (24 + 16 + 8) |

#### 消除二级标题内部间距 (Zero Title Margin)

**核心目标**：彻底切除二级菜单标题内部图标与文字之间的任何非标间距。

**物理约束**：
- 图标左边距：固定 24px
- 图标尺寸：固定 16px × 16px
- 图标右间距：固定 8px，**严禁出现任何额外的间距**
- 文字起始位：必须精准落在水平坐标 48px 处

#### 三级菜单 (Level 3)
| 属性 | 规范值 |
|-----|--------|
| 视觉约束 | 取消组件库默认缩进逻辑 |
| 强制偏移 | 文字起始位置必须与二级菜单对齐 |
| **文字起始坐标** | **48px** |

#### 四级菜单 (Level 4)
| 属性 | 规范值 |
|-----|--------|
| 对齐参照 | 相对于三级菜单缩进 12px |
| **文字起始坐标** | **60px** (48 + 12) |
| 圆点标识 | 4px × 4px 实心圆点 |
| 圆点位置 | 文字左侧 8px 处 |

#### 阅读线对齐示意
```
|<- 24px ->|<- 16px ->|<- 8px ->|
|          |   图标   |          | 文字开始 (48px)
           └────────────────────── 二级菜单文字起始线
                                     │
                                     ├─ 三级菜单文字 (48px)
                                     │   └─ 四级菜单圆点 (52px)
                                     │       └─ 四级菜单文字 (60px)
```

### 3.8 样式强制装配要点

以下为必须强制实现的视觉规范，无论使用何种技术栈均需确保：

1. **二级标题内部间距归零**：图标左边距 24px、右间距 8px，文字起始位 48px，严禁任何额外间距
2. **三级菜单对齐 48px 阅读线**：padding-left: 48px，取消组件库默认缩进
3. **四级菜单基于 48px 缩进 12px**：padding-left: 60px
4. **小标题不可交互**：cursor: default，user-select: none，pointer-events: none；悬浮/焦点/激活态均透明
5. **数据防泄密小标题特殊左间距**：padding-left: 48px
6. **选中态蓝色激活条**：3px 垂直蓝色边框 + 蓝色渐变底色（从左到右，20%蓝色透明度到透明）
7. **二级菜单选中态图标**：蓝绿渐变（#1C6EFF → #6DD405），文字白色 600 字重
8. **三级菜单未选中态**：12px/400/graphite-l10 (#BEC3CC)
9. **四级菜单圆点**：4px 实心圆，graphite-d10 (#6F7785)，与文字间距 8px

## 4. 交互与联动逻辑

### 4.1 状态记忆机制 (Status Memory)

#### 核心原则
当用户在顶部一级导航之间切换时，系统必须具备"回头看"的能力，而不是盲目重置。

#### 保存逻辑
当用户点击新的一级菜单 A 时，系统必须立即将当前菜单 B 的左侧选中项（末级 Key）保存到全局状态中。

**注意事项**：
- 只保存可点击的菜单项（有 key 属性的节点）
- 小标题（如"分析"、"策略"）不参与历史记录，因为它们不可点击

#### 恢复逻辑
进入新菜单 A 前，系统需检索历史记录：
- **有记录**：直接恢复并跳转至上次记录的末级菜单
- **无记录**：执行默认逻辑，激活该一级导航下的第一个叶子节点

#### 父级自动展开
无论是恢复历史还是激活首项，如果目标菜单是三级或四级，必须自动展开其所有父级目录，确保用户能看到清晰的导航路径。

### 4.2 首项激活

点击一级菜单时，如果没有历史记录，必须递归查找并自动跳转至该路径下的第一个最末级叶子节点。

**注意事项**：
- 查找时跳过小标题（如"分析"、"策略"），因为它们不是可点击的菜单项
- 小标题没有 key 属性，不参与历史记录的保存与恢复

### 4.3 二级菜单互斥展开逻辑

#### 核心原则
当用户在二级菜单之间切换时，系统必须自动收起上一个选中的二级菜单，避免左侧导航过长。

#### 实现逻辑
1. **展开状态管理**：维护当前展开的二级菜单 Key 列表
2. **互斥收起**：点击新的二级菜单时，自动移除其他已展开的二级菜单
3. **保留当前项**：只保留当前点击的二级菜单在展开状态
4. **排除小标题**：小标题（如"分析"、"策略"）不参与展开/收起逻辑，不影响菜单层级

#### 注意事项
- 该逻辑仅针对二级菜单（有子菜单的菜单项）
- 小标题（如"分析"、"策略"）不参与展开/收起逻辑，因为没有 children
- 三级及四级菜单不受影响，可以正常展开/收起
- 切换时保留当前选中的菜单项高亮状态

### 4.4 侧边栏收起交互

#### Hover 展开
在固定收起状态下，鼠标移入区域需自动临时展开，移开后恢复。

#### 固定状态
通过点击标题栏的收起/展开图标实现 240px 固定展开。
