本规范只定义 XDR 导航的视觉、结构、交互和数据规则，不提供任何具体代码实现。

生成代码时，必须以当前项目的技术栈 Skill 为准。

如果本规范中的设计描述与项目技术栈约束发生冲突，优先遵守项目技术栈约束。

本规范禁止包含：

### 分组标题规范

分组标题用于菜单分组，只起到视觉分割作用，不是具体的菜单项。

| 类型 | 字号 | 字重 | 颜色 | 左边距 | 高度 | 选中态 | 交互 |
|------|------|------|------|--------|------|--------|------|
| 二级分组标题 | 12px | 400 | var(--color-graphite-d20) (#5E6573) | 40px | 32px | 无 | 不可点击 |

**核心特性**：
- 不可点击，仅起视觉分隔作用
- 没有选中态，永远不会被标记为选中
- 不影响其他菜单的层级结构
- 不参与菜单的展开/收起逻辑
- 不参与菜单的历史记录保存与恢复
- 收起态下分组标题自动隐藏
- 悬浮态/焦点态/激活态均透明，无任何背景或颜色变化

## 4. 视觉状态红线 (Visual States)

### 4.1 选中态 (Selected)

| 属性 | 规范值 |
|-----|--------|
| 背景 | 蓝色渐变，从左侧 rgba(28,110,255,0.2) 渐变至右侧透明 |
| 左侧激活条 | 3px 垂直蓝色条，紧贴左边缘，不影响图标和文字的原始位置 |
| 文字颜色 | 纯白 #FFFFFF |
| 图标位置 | 不变（选中与未选中图标和文字位置完全一致） |
| 一级菜单字重 | 600（未选中为 400） |
| 二级菜单字重 | 600（未选中为 400） |

**适用范围**：
- 二级菜单可进入选中态
- 一级菜单如果没有包含子菜单（即本身为叶子节点），也可以进入选中态，此时图标变蓝（#1C6EFF）
- 一级菜单如果包含子菜单，则不进入选中态，而是触发父级高亮（见 4.2）
- 分组标题不可进入选中态

**重要约束**：
- 选中的蓝色条不能影响菜单图标和文字的位置
- 选中和未选中时候图标和文字的位置是不变的，不要产生右移

### 4.2 父级高亮 (Parent Active)

当一级菜单包含二级菜单且子菜单选中时，触发以下效果：

| 属性 | 规范值 |
|-----|--------|
| 一级菜单图标颜色 | #1C6EFF (blue) |
| 一级菜单文字颜色 | 保持原色（白色），不变色 |
| 一级菜单底色 | #1E232B |
| 同层级二级菜单底色 | #1E232B |
| 同层级二级分组标题底色 | #1E232B |

**重要约束**：
- 二级菜单被选中时，该二级菜单所在的一级菜单以及同一层级的二级菜单的底色改为 #1E232B
- 一级菜单仅图标变色（blue），文字保持白色不变

## 5. 交互与联动逻辑

### 5.1 点击展开与手风琴模式

| 规则 | 说明 |
|------|------|
| 点击展开 | 一级菜单必须为点击展开，不允许 hover 展开 |
| 手风琴模式 | 同一时间仅允许一个一级菜单展开，点击某个一级菜单时展开该项并收起其他一级菜单 |
| 选中联动 | 选中某个二级菜单后，仅保留其所属一级菜单展开，其他一级菜单全部收起，并显示当前二级菜单选中态 |
| 保证可见 | 确保选中的菜单能够直接被看到 |
| 互斥收起 | 点击新的一级菜单时自动收起其他已展开的一级菜单 |

**实现逻辑**：
1. 维护当前展开的一级菜单 Key 列表
2. 点击一级菜单时，替换为当前点击的菜单 Key（手风琴：仅保留一个）
3. 选中二级菜单时，自动展开其所属一级菜单，收起其他一级菜单
4. 排除小标题：小标题不参与展开/收起逻辑，因为没有子菜单

### 5.2 折叠交互

| 规则 | 说明 |
|------|------|
| 收起态宽度 | 54px |
| Hover 交互 | 鼠标 Hover 区域需临时悬浮展开完整菜单，浮层带阴影效果 |
| 恢复机制 | 移开后恢复 54px 收起态 |
| 收起态隐藏 | 文字、产品名称、用户名、展开箭头、分组标题全部隐藏 |
| 收起态图标 | 一级菜单图标水平居中，左边距 19px（(54-16)/2） |
| 收起态用户区 | 头像居中显示，取消左间距 |

## 6. 样式强制装配要点

以下为必须强制实现的视觉规范，无论使用何种技术栈均需确保：

1. **一级菜单图标位置**：左边距 16px，尺寸 16px × 16px，与文字间距 8px
2. **二级菜单圆点位置**：左边距 40px，尺寸 4px × 4px，与文字间距 8px
3. **二级分组标题左边距**：40px，不可点击，无选中态
4. **选中态蓝色激活条**：3px 垂直蓝色边框 + 蓝色渐变底色（从左到右，20%蓝色透明度到透明），必须不影响图标和文字的原始位置
5. **父级高亮**：子菜单选中时，一级菜单图标变蓝（#1C6EFF），文字不变色，同层级底色 #1E232B
6. **叶子节点选中**：无子菜单的一级菜单选中时，图标变蓝（#1C6EFF）
7. **一级菜单字号**：13px，选中字重 600，未选中字重 400
8. **二级菜单字号**：12px，选中字重 600，未选中字重 400
9. **分组标题颜色**：二级分组标题 var(--color-graphite-d20) (#5E6573)，悬浮/焦点/激活态均透明
10. **互斥展开**：同时只允许一个一级菜单展开
11. **收起态**：54px 宽度，图标居中，文字和分组标题隐藏
12. **菜单标签**：（GPT）和（推荐）使用蓝色标签展示，文字颜色 var(--color-blue)

## 8. 设计原则

| 原则 | 说明 |
|------|------|
| 深色一致性 | 侧边栏整体保持 #272E38 深色背景，所有状态变化在深色基础上叠加 |
| 坐标严格性 | 一级菜单图标 16px 左边距、二级菜单 40px 起始坐标必须严格执行 |
| 布局稳定性 | 选中态蓝色条不影响图标和文字的原始位置 |
| 互斥展开 | 同一时间只允许一个一级菜单展开（手风琴模式），不允许 hover 展开 |
| 父级联动 | 子菜单选中时父级图标变蓝 + 同层级底色变化，形成视觉联动 |
| 分组纯视觉 | 二级分组标题仅起视觉分割作用，不参与任何交互逻辑 |
| 标签识别 | （GPT）和（推荐）使用蓝色标签，帮助用户快速识别菜单特征 |

## 9. 注意事项

- 选中态蓝色条不能改变图标和文字的水平位置
- 一级菜单无子菜单时可直接进入选中态（图标变蓝 + 文字变白）
- 父级高亮仅图标变蓝，文字保持原色
- 同层级底色 #1E232B 仅在子菜单选中时触发
- 收起态 Hover 展开后移开鼠标自动恢复 54px
- 一级菜单必须点击展开，不允许 hover 展开
- 禁止同时展开多个一级菜单（手风琴模式）
- 选中二级菜单时必须自动展开其父级菜单，并收起其他一级菜单
- XDR 导航无一级分组标题，只有二级分组标题（左边距 40px），不可点击、无选中态
- 分组标题在收起态下自动隐藏
- 禁止选中态影响图标和文字的水平位置
- 禁止同时展开多个一级菜单- 禁止父级高亮时文字变色
- 禁止分组标题参与交互逻辑
- （GPT）和（推荐）标签使用蓝色文字展示，字号与所在菜单层级一致
