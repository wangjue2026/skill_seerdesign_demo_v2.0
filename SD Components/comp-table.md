本规范只定义表格组件和分页器组件的视觉、布局、交互和验收规则，不直接提供具体框架代码实现。

生成代码时，必须以当前项目的技术栈 Skill 为准。

如果本规范与项目技术栈约束发生冲突，优先遵守项目技术栈约束。

本规范禁止包含：
- React / Vue / Angular 代码示例
- JSX / TSX
- TypeScript 示例
- import 语句
- 具体组件库 API
- React render JSX
- Vue3 Composition API
- 具体组件库内部 class 选择器的全局覆盖

本规范中的 Table、Column、Selection、Pagination、Fixed Column 等名称仅表示设计语义，不代表必须使用某个框架或组件库的同名 API。

如果需要针对 Vue2 + ant-design-vue 1.7.8 生成代码，必须由 Vue2 技术栈 Skill 负责适配实现。

## 一、表格组件规范

### 1.1 核心规范

- **行高**：40px
- **表头高度**：32px
- **表头背景**：#EDF1F7 (graphite-l40)
- **表格无圆角**：四角均为直角
- **行高固定**：默认单行表格内容区行高为 40px，不因单元格内容过长自动撑高
- **横向溢出处理**：当列数量或列宽超过可视区域时，表格出现横向滚动条，不通过换行、压缩列宽或拉高行高来展示内容
- **操作列文字**：统一使用链接色 blue (#1C6EFF)，禁止使用其他颜色
- **内容区行分割线**：仅保留底部边框线，去除顶部边框线，避免相邻行上下边框叠加导致横向分割线变粗
- **更多列**：位于操作列之后，列宽固定 40px，表头展示省略号图标，内容区为空白单元格，不承载行级操作
- **操作列冻结**：有横向滚动条时，操作列要冻结
- **单元格左右间距**：12px
- **表头纵向分割线**：默认需要
- **表头文字不换行**
- **表头文字默认左对齐**，仅序号列和多选框列居中对齐

### 1.2 文字规范

| 区域 | 字号 | 字重 | 颜色 | 说明 |
|------|------|------|------|------|
| 表头文字 | 12px | 400 | #454C59 (graphite-d30) | 不换行 |
| 内容文字 | 12px | 400 | #2F3540 (graphite-d40) | 默认样式 |
| 可点击文字 | 12px | 400 | #1C6EFF (blue) | 链接色 |
| 可点击文字 hover | 12px | 400 | #4FA1FF (blue-l10) | 悬浮态 |
| 操作列文字 | 12px | 400 | #1C6EFF (blue) | 统一蓝色 |
| 序号文字 | 12px | 400 | #2F3540 (graphite-d40) | 居中 |

### 1.3 对齐规范

| 列类型 | 表头对齐 | 内容对齐 | 说明 |
|--------|---------|---------|------|
| 文字列 | 左对齐 | 左对齐 | 默认 |
| 数字列 | 右对齐 | 右对齐 | 金额、数量、百分比、流量、时长。编号、ID、SN、版本号、端口号等标识类字段即使包含数字，也按文本列处理，默认左对齐 |
| 序号列 | 居中 | 居中 | 固定宽度 40px |
| 多选框列 | 居中 | 居中 | 固定宽度 40px |
| 操作列 | 左对齐 | 左对齐 | 特殊场景可右对齐，但同列必须一致 |

### 1.4 实现约束与验收标准

#### 1.4.1 行高与文字样式

表格必须满足以下视觉要求：

- 表头高度固定为 32px
- 内容区单行行高固定为 40px
- 表头文字不允许换行
- 内容区单元格左右内间距为 12px
- 表头文字字号为 12px，字重 400，颜色 #454C59
- 内容文字字号为 12px，字重 400，颜色 #2F3540
- 可点击文字使用品牌蓝 #1C6EFF
- 可点击文字 hover 时可使用浅蓝 #4FA1FF

验收标准：
- 表头高度一致，无高低错位
- 内容行高一致，不因某列内容变化而抖动
- 表头文字不换行
- 内容文字与表头文字视觉层级清晰

#### 1.4.2 表格无圆角

表格本体不使用圆角。

要求：
- 表头左上角和右上角不出现圆角
- 表格内容区左下角和右下角不出现圆角
- 即使全局组件圆角为 2px，表格组件也必须保持直角

验收标准：
- 表格四角均为直角
- 表格嵌入页面卡片时，由外层卡片决定圆角，表格自身不额外添加圆角

#### 1.4.3 序号列

序号列用于展示行序号。

要求：
- 宽度固定为 40px
- 表头和内容均居中对齐
- 序号列不承载业务操作
- 序号数字使用普通内容文字样式

验收标准：
- 序号列宽度不随内容变化
- 序号在单元格内水平和垂直居中

#### 1.4.4 多选框列

多选框列用于批量选择。

要求：
- 宽度固定为 40px
- 表头全选框水平和垂直居中
- 内容区复选框水平和垂直居中
- 选中态使用品牌蓝 #1C6EFF
- 未选中态边框使用 #D3D7DE

验收标准：
- 表头全选框与内容复选框对齐
- 多选框列宽度不随表格滚动或内容变化

#### 1.4.5 表头纵向分割线

表头默认显示纵向分割线。

要求：
- 分割线只出现在表头区域
- 内容区严禁出现纵向分割线
- 分割线高度为 16px
- 分割线颜色为 #E1E5EB
- 分割线垂直居中
- 最后一列表头右侧不显示分割线

验收标准：
- 表头列之间有清晰但轻量的分割线
- 内容区列之间没有竖线
- 表头分割线不会造成表头文字错位

#### 1.4.6 数字列右对齐

数字列包括金额、数量、百分比、流量、时长等可计算数值。

**注意**：编号、ID、SN、版本号、端口号等标识类字段即使包含数字，也按文本列处理，默认左对齐。

要求：
- 数字列表头右对齐
- 数字列内容右对齐
- 同一列的表头与内容对齐方式必须一致

验收标准：
- 数字列的小数点或数值末尾视觉上形成整齐边界
- 表头与内容不存在左右错位
- 标识类字段正确使用左对齐

#### 1.4.7 操作列

操作列用于承载行级操作。

要求：
- 操作列文字统一使用品牌蓝 #1C6EFF
- 禁止在操作列中使用红色、灰色等其他操作文字颜色
- 操作之间仅使用 16px 水平间距，不使用竖线分隔符
- 操作列宽度需要根据操作数量明确设置
- 当表格出现横向滚动时，操作列必须冻结在右侧

验收标准：
- 所有操作文字颜色一致
- 操作项之间间距为 16px
- 不出现竖线分隔符
- 操作项可点击状态明确
- 横向滚动时操作列始终可见

#### 1.4.8 列对齐一致性

每一列必须保持表头和内容对齐方式一致。

要求：
- 文字列默认左对齐
- 数字列右对齐
- 序号列居中对齐
- 多选框列居中对齐
- 操作列默认左对齐，特殊场景可右对齐，但同列必须一致

验收标准：
- 不允许出现表头左对齐、内容右对齐的错位情况
- 不允许因为自定义渲染内容导致列对齐破坏

#### 1.4.9 单行与双行模式

表格默认使用单行模式。

单行模式要求：
- 内容区行高为 40px
- 单元格内容默认单行展示
- 超长文本可以省略，并通过 Tooltip 或详情入口展示完整内容

双行模式适用场景：
- 描述类字段较长
- 需要展示主副文本
- 需要展示名称 + 辅助说明

双行模式要求：
- 行高可扩展为 60px
- 主文本与副文本上下排列
- 主文本和副文本之间需要有明确层级
- 双行模式必须整行统一高度，避免同一行不同列高度不一致

验收标准：
- 默认表格不因长文本自动撑高
- 双行模式下行高统一
- 单行表格和双行表格不能混用导致视觉跳动

#### 1.4.10 横向滚动与操作列冻结

当列数量或列宽超过可视区域时，表格必须支持横向滚动。

要求：
- 不允许通过压缩列宽导致内容不可读
- 不允许通过增加行高来展示横向内容
- 需要明确设置每列宽度
- 表格横向滚动时，操作列冻结在右侧
- 冻结操作列需要有左侧分隔线，区分固定区与滚动区
- 冻结列背景必须覆盖滚动内容，避免文字重叠

验收标准：
- 横向滚动条出现时，所有列内容都可访问
- 操作列始终保持可见
- 冻结列与滚动区域之间边界清晰
- 冻结列不会透出底层滚动文字

#### 1.4.11 表格更多列 / 列设置列

更多列用于表头级更多配置、列设置、扩展能力入口，不表示行级操作，不参与每一行的数据操作。

**位置**：
- 默认位于表格最右侧，位于操作列之后
- 如果表格存在横向滚动，更多列可与操作列一起固定在右侧

**视觉规范**：
- 列宽固定为 40px
- 表头单元格中展示横向省略号图标
- 图标水平、垂直居中
- 表格内容区对应单元格为空白
- 内容区单元格不得展示省略号、按钮、图标或文字
- 内容区空白单元格仍然保留列宽，用于保证表头和内容区列宽一致

**交互规范**：
- 更多图标只出现在表头
- 如支持列设置，点击表头更多图标打开列设置、密度设置或表格配置菜单
- 行内容区的空白单元格不可点击
- 不得将该列实现为行级"更多操作"

**隐藏规则**：
- 常规页面表格默认展示更多列
- 弹窗、抽屉、卡片内的小型表格，如果字段较少、无列设置需求，可以隐藏更多列
- 是否隐藏由表格场景控制，不由单行数据控制

**验收标准**：
- 更多列列宽固定为 40px
- 表头省略号图标水平垂直居中
- 内容区单元格为空白，不得承载操作
- 横向滚动时更多列与操作列一起冻结

#### 1.4.12 滚动表头样式隔离与空白行防范

在使用横向/纵向滚动（如 `scroll` 属性）时，大部分前端表格组件库（如 Ant Design 等）会使用额外的辅助 DOM 元素来处理列宽测量和对齐：
1. **隐藏表头**：在内容表格（Body Table）中，为了确保每一列的宽度能与上方表头对齐，组件库会默认生成一个高度为 0 且不可见的 `thead`（通常带有 `visibility: hidden; height: 0;` 等属性）。
2. **测量行（Measure Row）**：在内容表格的 `tbody` 顶部，为了精确测量各列宽度，组件库会动态生成一行高度为 0 且无边距的列宽对齐行（如 `tr.ant-table-measure-row`）。

如果开发者直接对表头单元格或内容单元格的选择器进行样式定制，特别是通过全局覆盖且包含 `!important` 的 `height` 或 `padding` 属性时，样式会错误地继承或强行覆盖到这些本应完全折叠的辅助元素上，导致其被强行撑开，最终在表头与第一行数据之间渲染出一个巨大的**空白行**。

**规范与要求**：
- **精准样式作用域隔离**：在定制表头单元格的高度、内边距等任何可能撑开空间的属性时，严禁使用范围过大的全局类覆盖选择器（例如 `:deep(.ant-table-thead > tr > th)`）。必须将定制选择器限制在可见的表头容器中，如 `:deep(.ant-table-header .ant-table-thead > tr > th)` 或 `:deep(.ant-table-content .ant-table-thead > tr > th)`。
- **内容行避开测量行（防范空白行关键！）**：在定制表格内容区单元格高度与内边距时（如设置行高为 40px），严禁使用普适型的 `tr > td` 覆盖选择器（例如 `:deep(.ant-table-tbody > tr > td)`），这会强行撑开隐藏的测量行。**必须**使用排除伪类将测量行过滤掉，或采用针对性的选择器：
  - 推荐选择器：`:deep(.ant-table-tbody > tr:not(.ant-table-measure-row) > td)`
  - 推荐 hover 态：`:deep(.ant-table-tbody > tr:not(.ant-table-measure-row):hover > td)`
  - 兜底重置规则（若因其他库引入产生漏网之鱼）：
    ```css
    :deep(.ant-table-tbody > tr.ant-table-measure-row > td) {
      height: 0 !important;
      padding: 0 !important;
      line-height: 0 !important;
      border: none !important;
    }
    ```
- **谨慎滥用 `!important` 声明**：针对表头高度、边距等空间撑开类的属性，应尽量避免滥用 `!important` 修饰符，以便让组件库原生的内联隐藏重置样式（例如内联 `height: 0px`）能够在隐藏 `thead` 或测量行子节点上自然生效。
- **验收排查标准**：凡是使用了滚动展示（`scroll`）的表格页面，在验收时必须特别关注并排查表头下方与第一行数据之间是否存在无内容占位的空白高度，必须确保隐藏 `thead` 和测量行的高度折叠为 0，防止页面布局受损。

## 二、分页器组件规范

本组件遵循 [全局原子规范与间距规范](design-atomic-spacing.md) 中的设计原则。

### 2.1 组件构成

| 序号 | 名称 | 说明 |
|-----|------|------|
| 1 | 总数（可选） | 显示数据总条目数。默认选项内容为 10、20、50、100，默认选项为 20 条。选项内容和默认选项业务可定义。若与页面其他内容重复，可去掉。 |
| 2 | 页码 & 上下翻页 | 分页基础功能。 |
| 3 | 单页显示数量设置（可选） | 仅当页面数很少时可不提供。 |
| 4 | 快速跳转（可选） | 输入框尺寸为默认可填入 3 个数字。若通常数据量达 10000，可增宽输入框。 |

### 2.2 组件类型

#### 常规型分页
- 适用场景：一般情况下默认使用
- 包含内容（依次）：总数（共 XX 项）+ 页码 & 上下翻页 + 单页显示数量设置 + 快速跳转（前往 X 页）

#### 精简型分页
- 适用场景：空间较小场景，如卡片中表格、穿梭框中表格
- 包含内容（依次）：总数 + 页码 & 简洁上下翻页

### 2.3 视觉规范

- 默认右对齐
- 默认无上下分割线

| 元素 | 颜色规范 | 说明 |
|-----|---------|------|
| 文字 | var(--color-graphite-d10) | 辅助文字色 |
| 数字 | var(--color-graphite-d40) | 正文色 |
| 当前页码背景 | var(--color-blue)，直径 24px 圆形 | 当前选中页 |
| 当前页码文字 | var(--color-white) | 当前选中页文字 |
| 页码悬浮边框 | var(--color-blue) | 悬浮态 |
| 页码点击边框 | var(--color-blue-d10) | 点击态 |
| 禁用态文字 | var(--color-graphite-l10) | 不可点击 |

### 2.4 尺寸规范

| 尺寸 | 高度 | 适用场景 |
|-----|------|---------|
| 默认 | 40px | 一般场景 |
| 更高 | 56px | 需要更大尺寸的场景 |

### 2.5 快速跳转输入框
- 宽度：48px（默认可填入 3 个数字）
- 高度：32px
- 边框色：var(--color-graphite-l20)
- 圆角：2px
- 字号：12px
- 文字颜色：var(--color-graphite-d40)
- 悬浮边框色：var(--color-blue)
- 激活边框色：var(--color-blue)，无阴影

### 2.6 禁用状态
- 禁用态页码文字颜色：var(--color-graphite-l10)
- 禁用态上下翻页颜色：var(--color-graphite-l10)
- 禁用态光标：not-allowed

### 2.7 验收标准
- 分页组件默认右对齐
- 默认无上下分割线
- 分页组件字号为 12px
- 当前页码背景为蓝色圆形（直径 24px），文字为白色
- 禁用态视觉可区分
- 快速跳转输入框尺寸和样式正确
- 分页区域高度一致
- 样式覆盖必须限制在当前页面作用域内
