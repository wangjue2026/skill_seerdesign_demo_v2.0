# 描述列表 Descriptions

## 1 使用声明

本规范只定义描述列表的视觉结构、布局规则和展示行为。

- 描述列表仅用于静态详情展示，不包含表单校验和交互逻辑
- 不绑定具体框架或组件库
- 与项目技术栈 Skill 冲突时，优先遵守项目技术栈约束

## 2 组件定义

描述列表（Descriptions）是一种静态信息展示组件，由 **Label**（字段标题）和 **Value**（字段内容）组成。

- Value 为纯静态文本，默认不可编辑
- 无需表单校验规则
- 无需交互逻辑（如失焦验证、输入提示等）
- 结构上与表单相似，但只承载阅读信息

## 3 基础视觉规范

| 属性 | 值 |
|------|-----|
| 文字字号 | 12px |
| 文字字重 | 400（常规体） |
| 行高 | 20px |
| 每行间距 | 6px |
| Label 颜色 | graphite-d10 |
| Value 颜色 | graphite-d40 |
| Label 与 Value 间距 | 固定 16px |

## 4 描述项结构

每个描述项由 Label 和 Value 组成，默认水平同在一行。

### 4.1 单个描述项

当组件只有一个描述项时：

- Label 宽度由自身文字内容决定（不参与最长匹配计算）
- Value 宽度自适应剩余空间

### 4.2 多个描述项

当组件包含多个描述项时：

- 所有描述项的 Label 宽度按当前 Descriptions 实例内同一列的最长 Label 统一计算
- 同一列的所有 Label 统一宽度
- 同一列的所有 Label 文字左对齐
- Value 左边缘与统一后的 Label 区域右边缘对齐

## 5 Label 与 Value 对齐规则

### 5.1 同行对齐

Label 与 Value 默认水平排列在同一行，Value 左边缘与 Label 区域右边缘间距为 16px。

### 5.2 多行 Value 时 Label 对齐

当 Value 内容超过一行发生换行时：

- Label 与 Value 第一行顶部对齐（不是垂直居中）
- 当前描述项高度随 Value 内容自适应增长

### 5.3 禁止规则

- 禁止让 Label 与 Value 垂直居中（无论 Value 单行还是多行）
- 禁止将 Label 区域宽度压缩到仅包裹自身文字（多描述项场景下必须统一宽度）

## 6 单列布局

一组 Label + Value 占满整行宽度，适用于信息项较少或需要完整展示的场景。

## 7 多列布局

### 7.1 布局策略

多列布局采用横向优先排布策略：

- Label + Value 整体作为一个描述项
- 多个描述项可以在同一行横向排列（多列），也可以换行形成多行
- 列间距不是固定值

### 7.2 列数确定规则

列数按以下优先级确定：

**第一优先级：业务指定**
- 如果业务或调用方明确指定了列数，必须按业务指定执行
- AI 不得覆盖业务指定的列数

**第二优先级：AI 自动判断**
- 未指定列数时，AI 根据描述项数量、容器宽度和 Value 长度综合判断
- 默认优先使用横向空间，再使用纵向空间

### 7.3 自动列数推断规则

综合考虑以下因素进行判断：

- Label 文本长度
- Value 文本长度
- Label 与 Value 间距（固定 16px）
- 描述项之间的列间距（档位见 7.4）
- 当前容器宽度

**描述项数量较少时**（2～3 个）：

- 优先尝试在一行内横向展示完
- 如果一行展示会导致 Value 过窄、文本频繁换行或内容不可读，则减少每行数量（降为 2 列或单列）

**描述项数量较多时**（6 个及以上）：

- 默认优先使用 3 列布局
- 例如 6 个描述项：默认 3 列，每列 2 个
- 如果容器空间不足以支持 3 列，则降级为 2 列
- 如果 2 列仍导致 Value 过窄或内容不可读，则降级为单列

**窄容器或长 Value 场景**：

- 抽屉、弹窗、侧边配置面板等横向空间较窄的容器，默认优先 2 列或单列
- 如果 Value 内容较长（如 URL、路径、描述文本、说明性内容、状态原因等），不应强行多列展示
- Value 较长时，优先保证 Value 可读性，必要时降为 2 列或单列
- 不得为了凑满特定列数而压缩 Value 到不可读

**横向优先原则**：

- 未指定列数时，应先判断当前描述项是否可以在容器内横向排布
- 能横向展示且内容可读时，优先横向展示
- 横向空间不足时，再增加纵向行数
- 不得一开始就机械地全部按单列垂直排列
- 也不得在空间不足时机械地强行多列展示

**推荐默认策略**：

| 描述项数量 | 推荐布局 |
|-----------|---------|
| 1 个 | 单行展示 |
| 2～3 个 | 优先一行（2 列或 3 列）；空间不足时降为 2 列或单列 |
| 4～5 个 | 优先 2 列或 3 列（根据空间和 Value 长度判断） |
| 6 个及以上 | 默认优先 3 列；空间不足时降为 2 列，再不足时降为单列 |
| 窄容器 / 长 Value | 默认优先 2 列或单列 |

### 7.4 列间距档位

列间距不是固定 16px，而是在以下档位中选择：

| 档位 | 适用场景 |
|------|---------|
| 48px | 空间充足且视觉需要更舒展时 |
| 32px | 默认推荐 |
| 24px | 空间略有不足时降级 |
| 16px | 最小间距，不得小于此值 |

**列间距选择原则**：

1. 优先使用 32px
2. 空间充足且视觉需要更舒展时，可使用 48px
3. 空间不足时，降级为 24px
4. 再不足时，降级为 16px
5. 若 16px 仍不足，应减少列数或降级为单列，而不是继续压缩间距、Label 或 Value

### 7.5 列间距禁止规则

- 禁止使用 `justify-content: space-between` 导致列间距被无限拉大
- 容器剩余空间应优先分配给 Value 文本区域，而不是无限放大列间距
- 列间距不得大于 Label + Value 这一列的宽度

### 7.6 多列示意

```
| Label1  Value1 | 列间距 | Label2  Value2 | 列间距 | Label3  Value3 |
```

列间距从档位 16px / 24px / 32px / 48px 中选择，不得使用 space-between 拉大。

## 8 换行与长文本规则

### 8.1 Value 换行

- Value 默认允许自动换行
- 长文本、URL、连续字符等应允许断行，避免撑破容器
- 换行后该描述项高度自动适配

### 8.2 Label 换行

- Label 默认不换行
- Label 参与最长 Label 宽度计算
- 不得因为 Label 较长而随意压缩或换行
- 如果业务需要限制 Label 最大宽度，必须由具体业务另行指定

### 8.3 连续字符断行

- URL、长数字、连续英文字符等应允许 word-break 断行
- 不得因为长连续字符未换行而撑破容器宽度

## 9 空值展示规则

当 Value 为空、null、undefined 时：

- 默认显示 "—"
- 不得因为 Value 为空而隐藏该描述项
- 除非业务明确要求隐藏空值字段

## 10 响应式降级规则

当容器宽度不足时：

- 列间距优先降档：48px → 32px → 24px → 16px
- 列间距降到 16px 后仍不足时，减少列数
- 直至降级为单列
- 不得压缩 Label、Value 或列间距到不可读
- 降级后仍需遵循：
  - Label 与 Value 间距 16px
  - 每行间距 6px
  - 文字 12px / 400 / 20px
  - Label 与 Value 第一行顶部对齐（多行 Value 时）

## 11 禁止项

- 禁止在描述列表中直接使用表单控件（如输入框、下拉选择器等），交互操作应跳转到编辑页
- 禁止让 Label 与 Value 垂直居中对齐
- 禁止在多描述项场景下让各 Label 独立宽度
- 禁止 Value 为空时隐藏描述项（除非业务明确要求）
- 禁止将长文本设为 overflow: hidden 或 white-space: nowrap 导致文字截断
- 禁止使用 space-between 拉大列间距
- 禁止列间距小于 16px

## 12 验收标准

### 视觉基础

- [ ] 文字字号为 12px，字重 400，行高 20px
- [ ] 每行间距 6px
- [ ] Label 与 Value 间距 16px
- [ ] Label 颜色 graphite-d10，Value 颜色 graphite-d40

### Label 与 Value 对齐

- [ ] Label 与 Value 同行水平排列
- [ ] 多行 Value 时，Label 与第一行顶部对齐（非垂直居中）
- [ ] 多描述项时，同列 Label 宽度统一，文字左对齐

### 多列布局

- [ ] 列间距使用档位 16px / 24px / 32px / 48px，不使用 space-between
- [ ] 列间距最小 16px，不得小于 16px
- [ ] 业务指定列数时按指定执行；未指定时按自动推断规则
- [ ] 窄容器或长 Value 场景默认优先 2 列或单列

### 换行与长文本

- [ ] Value 允许自动换行，不撑破容器
- [ ] 长 URL、连续字符允许断行
- [ ] Label 默认不换行，参与最长 Label 宽度计算

### 空值

- [ ] 空值默认显示 "—"
- [ ] 空值描述项正常显示，不被隐藏

### 响应式

- [ ] 容器宽度不足时，列间距优先降档
- [ ] 降档到 16px 仍不足时，减少列数直至单列
- [ ] 降级后间距和尺寸规则不变
