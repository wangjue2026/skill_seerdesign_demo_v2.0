# SeerDesign 表单规范

## ⚠️ 高频违规速查（阅读正文前必看）

> AI 在生成表单代码前，必须先逐行核对此表，再阅读下方完整规范。

| 规则 | ✅ 正确 | ❌ 常见错误 |
|------|--------|------------|
| 主按钮颜色（表单 / 弹窗 / 抽屉内） | `var(--color-blue)` — **任何场景的主按钮都是蓝色** | 用红色作为危险操作的主按钮 |
| 表单行基准高度 | `32px` | 随内容自动撑高 |
| Label 字号 | `12px`，颜色 `#6F7785`（graphite-d10） | 字号偏大或颜色过深 |
| Label 与控件间距 | `16px` | 使用 `8px` 或 `12px` |
| 控件高度 | `32px`（输入框、选择器、日期选择器等） | 控件高度不统一或未锁定 |
| 表单一级标题高度 | `32px`，字号 `14px`，字重 `600` | 使用 `<h3>` 等自带 margin 的 HTML 标题标签 |
| 蓝色竖条定位方式 | `position: absolute`，**不占用横向空间** | 占用横向空间导致有竖条标题的文字整体右移 |
| 蓝色竖条尺寸 | `2px × 12px`，在 32px 行高内垂直居中 | 尺寸不对或未垂直居中 |
| 有竖条与无竖条标题的文字对齐 | **左边缘像素级对齐**（竖条向左绝对定位偏移） | 有竖条的标题文字比无竖条的缩进或右移 |
| 首个一级标题上边距 | `0`（不额外增加） | 加了 `margin-top: 16px` 导致顶部多余空白 |
| 后续一级标题与上方表单项间距 | `16px` | 使用 `8px` |
| 标题与下方表单项间距 | `8px` | 使用 `16px` |
| 表单容器宽度（抽屉 / 弹窗内） | 充满内容区宽度，控件右侧不留空白 | 设为固定 `560px` 等导致右侧大量留白 |

## 使用声明

本规范只定义 SeerDesign 表单的视觉、布局、状态和交互规则，不提供具体框架代码实现。

生成代码时，必须以当前项目的技术栈 Skill 为准。

如果本规范与项目技术栈约束发生冲突，优先遵守项目技术栈约束。

本规范禁止包含：
- React / Vue / Angular 代码示例
- JSX / TSX
- TypeScript 示例
- import 语句
- 具体组件库 API
- 具体组件库内部 class 选择器
- React Hooks
- Vue3 Composition API

本规范中的 Form、Input、Select、Password、Search 等名称仅表示设计语义，不代表必须使用某个框架或组件库 of 同名组件。

## AI 使用原则

- 生成表单代码前，先判断表单使用场景：页面内、弹窗、抽屉、关联配置面板
- 再判断布局类型：水平布局（默认）、句式配置项
- 再应用表单项、控件、状态、提示、辅助图标规则
- 不得随意把表单宽度设为 100%
- 不得为了布局方便破坏 Label、控件、辅助列、提示文案的对齐规则

## 5.1 表单项规范

### 5.1.1 基础样式
- **行高**：32px
- **字号**：12px
- **字重**：400

### 5.1.2 Label 规范
- **字号**：12px
- **字重**：400
- **颜色**：辅助色 graphite-d10 (#6F7785)
- **间距**：Label 与输入框间距 16px

### 5.1.3 必填标识
- **标识符号**：红色星号 *
- **位置**：Label 文字前
- **颜色**：var(--color-red) (#F52727)
- **间距**：星号与 Label 文字间距 4px
- **对齐补齐规则**：为了确保在视觉上是文字与文字左对齐（而不是让非必填项的文字去对齐必填项的星号），必须采用“占位不显示”的处理方式：非必填项的 Label 左侧也必须预留一个与星号等宽的隐形占位空间（星号本身不显示），从而确保所有 Label 的首个文字在垂直方向上像素级对齐。
- **弹性容器实现策略**：为确保跨浏览器排版下首个字体的绝对对齐，必须将星号包装在固定的 `10px` 宽弹性容器中（如 `<span class="w-2.5 text-center text-risk mr-1 flex-shrink-0">*</span>`），非必填项占位符则使用同等宽度的空容器（如 `<span class="w-2.5 mr-1 flex-shrink-0"></span>`），通过容器宽度的一致性强制实现字体左对齐。

### 5.1.4 错误提示
- **字号**：12px
- **颜色**：red-d10 (#CF171D)
- **位置**：输入框下方
- **间距**：输入框与错误提示间距 4px

### 5.1.5 禁用状态
- **背景色**：graphite-l30 (#E1E5EB)
- **边框色**：graphite-l20 (#D3D7DE)
- **文字颜色**：graphite (#A1A7B3)

### 5.1.6 表单容器宽度规则
表单整体宽度根据使用场景决定，不得一刀切设为 100%。

#### 1) 抽屉 / 弹窗 / 关联配置面板内
- 表单从 Label 文字到控件右边缘（含小 i 辅助列）的整体单行宽度，必须充满所在容器的内容区
- 不得在控件右侧留出空白
- 表单宽度 = 容器内容区宽度

#### 2) 页面内独立表单
- 表单最大宽度按页面中最宽的控件类型而定，不得充满页面内容卡片宽度
- 推荐最大宽度递进档位：560px / 640px / 720px / 800px / 960px
- 普通输入框、选择器：560px 或 640px
- 穿梭框、内嵌表格、复合组件：720px / 800px / 960px
- 表单控件右边缘不得超出所选档位宽度

## 5.2 布局模式

### 5.2.1 默认布局
- **行间距**：8px
- **列间距**：16px

### 5.2.2 垂直布局
- **表单项间距**：8px
- **Label 与输入框间距**：8px

### 5.2.3 水平布局
- **对齐方式**：Label 采用左对齐（Left Aligned）。
- **Label 宽度**：统一设定固定宽度（如 100px 或 120px），确保整列对齐。
- **Label 与输入框间距**：固定 16px。
- **输入框宽度**：根据容器宽度自适应调整，最小宽度建议为 480px，以减少大尺寸容器（如抽屉）中的视觉留白。
- **上下间距**：各表单项之间保持 8px 的垂直间距。

## 5.3 实现要点

### 5.3.1 最长匹配原则

在水平表单中，Label 区域需要根据当前表单内最长 Label 文本自动确定宽度。

要求：
- 同一表单区域内所有 Label 宽度保持一致
- Label 文字左对齐
- Label 与输入控件之间保持 16px 间距
- 输入控件填充剩余宽度

### 5.3.2 输入控件样式

所有输入类控件需要满足以下视觉要求：
- 高度为 32px
- 字号为 12px
- 字重为 400
- 圆角为 2px
- Placeholder 字号为 12px
- Placeholder 颜色为 graphite (#A1A7B3)

适用控件包括：
- 文本输入框
- 密码输入框
- 下拉选择器
- 数字输入框
- 日期选择器
- 搜索输入框

### 5.3.3 错误提示样式

当表单项校验失败时：
- 错误提示显示在输入控件下方
- 错误提示与输入控件之间间距为 4px
- 错误提示字号为 12px
- 错误提示颜色为 red-d10 (#CF171D)
- 错误状态下输入控件边框颜色为 red-d10 (#CF171D)
- 聚焦时不出现额外的发光阴影

### 5.3.4 禁用状态样式

当表单项处于禁用状态时：
- 背景色为 graphite-l30 (#E1E5EB)
- 边框色为 graphite-l20 (#D3D7DE)
- 文字颜色为 graphite (#A1A7B3)
- 鼠标不可编辑
- 禁用态与只读态需要在视觉上可区分

### 5.3.5 表单控件对齐规则

1. 表单普通行的基准高度为 32px
2. Label 区域高度为 32px，文字垂直居中
3. 右侧控件区域最小高度为 32px
4. 输入框、选择器、日期选择器、数字输入框等控件本体高度为 32px
5. 开关、单选框、复选框等控件本体高度不足 32px 时，不强制拉伸控件本体
6. 低高度控件必须放在 32px 高度的控件容器中，并在容器内垂直居中
7. 禁止因为 Switch / Radio / Checkbox 高度不足导致其相对 Label 上移或下移

### 5.3.6 右侧辅助图标规则

1. 表单项右侧允许出现说明图标，例如小 i 图标
2. 说明图标属于表单项布局的一部分，占用横向空间，不允许绝对定位覆盖在控件上
3. 说明图标尺寸建议为 14px 或 16px，具体以图标规范为准
4. 说明图标与输入控件 / 选择控件之间的间距为 8px
5. 说明图标右边缘必须对齐表单内容区右边缘
6. 在抽屉或弹窗中，内容区左右 padding 为 32px，因此说明图标右边缘距离抽屉 / 弹窗外边缘为 32px
7. 同一表单区域内，只要任一表单项存在右侧说明图标，则该区域所有表单项都必须预留统一的右侧辅助列
8. 没有说明图标的表单项，也需要保留同等宽度的占位区域
9. 输入框、选择框、日期选择器、数字输入框等控件的右边缘必须在同一条竖线上
10. 不允许有说明图标的字段变短、无说明图标的字段变长，导致控件右边缘不对齐

右侧控件区域结构：
```
FormItem
 ├─ Label
 └─ ControlArea
     ├─ ControlMain
     │   └─ Input / Select / Radio / Switch / Checkbox
     └─ ControlSuffix
         └─ InfoIcon 或占位
```

- ControlMain：真正的输入控件区域
- ControlSuffix：右侧说明图标列

禁止把小 i 图标用 `position: absolute` 放置，否则后续输入框宽度、hover 区域、校验状态都会受影响。

### 5.3.7 表单项说明文字规则

- 说明文字显示在控件行下方
- 说明文字左边缘与右侧主控件左边缘对齐
- 说明文字与控件行间距为 4px
- 说明文字字号 12px，颜色 var(--color-graphite)
- 说明文字不参与 32px 控件行高度计算，允许表单项高度自适应

### 5.3.8 表单提示条

表单提示条分为字段级提示条和全局提示条。

#### 字段级提示条
适用场景：
- 用于解释某一个字段、某一个 Label 或某一个控件
- 例如：开启备份后，将在备份完成时自动结束
- 例如：存储空间由深信服提供

布局规则：
- 字段级提示条必须位于当前表单项的控件区域下方
- 字段级提示条左边缘与当前字段的输入框 / 选择框左边缘对齐
- 字段级提示条右边缘与当前字段的输入框 / 选择框右边缘对齐
- 字段级提示条不得横跨 Label 区域
- 字段级提示条与上方控件行间距为 4px
- 字段级提示条不参与普通控件 32px 高度计算，表单项高度允许自适应

右侧辅助列规则：
- 如果当前表单区域没有右侧小 i 图标列，字段级提示条拉通到控件区域右边缘
- 如果当前表单区域存在右侧小 i 图标列，字段级提示条需要与输入框 / 选择框一样扣除小 i 列宽和 8px 间距
- 没有小 i 的字段也要保留右侧辅助列占位，保证所有输入框、选择框和字段级提示条右边缘一致
- 字段级提示条内部的信息图标不等同于表单右侧小 i，不参与右侧辅助列计算

#### 全局提示条
适用场景：
- 用于解释整个抽屉、弹窗、表单或配置区域
- 与单个字段无绑定关系

布局规则：
- 全局提示条可以横跨整个内容区
- 在抽屉或弹窗内，左右边缘跟随内容区 32px padding
- 不受 Label 宽度、小 i 辅助列影响
- 不得插入到某个字段的控件区域内

### 5.3.9 句式配置项 / 自然语言配置项

适用场景：
- 配置项需要用自然语言表达；
- 一句话中嵌入输入框、数字输入框、下拉选择器、单位、变量值等组件；
- 常见于策略、限制、阈值、保护设置等配置场景。

结构：
Label + 句式配置内容区

句式配置内容区由若干片段组成：
- 文本片段；
- 输入控件片段；
- 单位片段；
- 强调文本片段；
- 标点片段。

布局规则：
1. 句式配置内容区位于表单控件区域内，不横跨 Label 区域。
2. 句式配置内容区使用横向流式布局。
3. 文字片段与组件片段之间间距为 8px。
4. 组件片段与组件片段之间间距为 8px。
5. 组件片段与单位文字之间间距为 8px。
6. 内容超出一行时允许自然换行。
7. 换行只能在片段之间自然换行，不允许输入框、选择器本身被压缩到不可用。
8. 每一行的基准高度为 32px。
9. 文字片段必须放在 32px 高度容器中垂直居中。
10. 输入框、数字输入框、选择器高度为 32px。
11. 句式配置项整体高度允许随换行自适应。
12. 句式配置项与下一个表单项之间仍遵循表单项间距 8px。
13. 所有的片段采用左对齐的方式，整个表单要看起来是左对齐的样式。
14. 所有的片段是左右结构，左文本标题右组件内容，严禁出现 Ant Design 默认的上标题下组件（Vertical）布局。
15. 视觉基准线：非必填项也必须在左侧预留星号占位空间（不显示星号），确保文字与文字在左侧垂直像素级对齐。

控件宽度要求：
- 数字输入框最小宽度为 80px。
- 短文本输入框最小宽度为 100px。
- 下拉选择器最小宽度为 100px。
- **通用表单配置项宽度**：在抽屉或大容器中，配置内容项的最小宽度应设为 480px，并随容器宽度等比增加，以减少视觉留白。
- 带单位的输入框，输入区域最小宽度为 50px，单位区域不压缩。
- 当容器宽度不足时，优先让文字换行，不压缩控件宽度。
- 不允许出现输入框、选择器过窄导致内容不可读。

### 5.3.10 非输入类控件对齐规则

适用控件：
- 按钮、复选框组、单选框组、开关、链接按钮、纯文本操作项等。

规则：
- 非输入类控件仍然位于当前表单项的 ControlMain 内。
- ControlMain 左边缘必须与普通输入框、选择器的左边缘对齐。
- 非输入类控件默认从 ControlMain 左边缘开始排列，不得在 ControlMain 内水平居中。
- 多个复选框、多个单选框、多个按钮横向排列时，也必须从 ControlMain 左边缘开始排列。
- 控件之间间距按组件规范或业务规则处理，不得通过 `space-between` 或 `center` 拉开。
- 按钮、复选框、单选框、开关等控件本体不需要被拉伸到填满 ControlMain。
- 如果控件本体高度不足 32px，必须放在 32px 高度控件容器内垂直居中，但水平方向仍然左对齐。

### 5.3.11 按钮类控件规则

Label 对应的控件区域中，如果包含按钮且按钮独占一行：
- 按钮不得被拉宽充满内容区。
- 必须保留按钮自身宽度和样式。
- 例如"连通性测试"按钮，按普通按钮宽度渲染，不得按输入控件规则拉伸至充满控件区域。

### 5.3.12 复合配置项 / 联动控件行

当一个 Label 对应多个联动控件时，例如"执行周期 = 周期选择器 + 周几/日期选择器 + 时间选择器"，这些控件必须作为同一个表单项的控件区域处理，不得拆成多个表单项。

#### 布局规则
- 多个控件必须放在同一个 ControlMain 内，整体宽度填满当前控件区。
- 控件之间间距固定为 8px。
- 控件行使用横向 flex 布局。
- 不得使用 `space-between` 拉开控件。
- 不得让任意一个子控件单独设置 `width: 100%` 后再追加其他控件。
- 不得让右侧控件被挤出控件区。

#### 宽度分配规则
复合控件分为两类：
1. **固定宽度控件**：适用于时间选择器、日期选择器、短按钮、单位文本
   - 不得被压缩，也不得被拉伸
   - 推荐使用 `flex: 0 0 auto` 或指定固定宽度
   - 例如时间选择器可使用 88px / 96px
2. **自适应控件**：适用于周期选择器、多选选择器、文本输入框、较长下拉选择器
   - 占用剩余空间
   - 推荐使用 `flex: 1 1 0`
   - 必须允许内容省略、Tag 折叠或内部滚动，不能把其他控件挤出

#### 执行周期示例
| 场景 | 布局 |
|------|------|
| 每天 | 周期选择器（自适应） + 时间选择器（固定宽度） |
| 每周 | 周期选择器（固定宽度） + 周几多选（自适应） + 时间选择器（固定宽度） |
| 每月 | 周期选择器（固定宽度） + 日期多选（自适应） + 时间选择器（固定宽度） |

#### 推荐实现
```css
.CompositeControlRow {
  display: flex;
  align-items: center;
  width: 100%;
  gap: 8px;
  min-width: 0;
}

.CompositeControlRow .固定宽度控件 {
  flex: 0 0 auto;
}

.CompositeControlRow .自适应控件 {
  flex: 1 1 0;
  min-width: 0;
}
```

## 5.4 表单一级标题

表单一级标题用于表单内部的一级分组，帮助用户理解当前表单区域的配置含义。

表单一级标题是独立的标题栏，不是表单项 Label，也不是普通 h1 / h2 / h3 标题。不得使用 Typography.Title 或其他带默认 margin 的标题组件实现；如使用，必须显式清除所有 margin、padding 和默认 line-height。

### 样式类型
表单一级标题支持两种样式：
1. 纯文字标题
2. 蓝色竖条标题

### 通用视觉规范
- 标题栏高度：32px
- 标题字号：14px
- 标题字重：600
- 标题颜色：var(--color-graphite-d40)
- 标题文字垂直居中
- 标题栏无背景色
- 标题栏无边框
- 首个标题不额外增加 top margin
- 表单一级标题自身不额外增加上下 padding
- 表单一级标题高度包含文字和竖条的垂直居中区域

### 蓝色竖条标题
- 竖条尺寸：2px × 12px
- 竖条圆角：0.2px
- 竖条颜色：var(--color-blue)
- 竖条在 32px 标题栏内垂直居中
- 竖条与标题文字间距：4px
- 竖条为装饰性标识，不参与标题文字左侧布局间距。有竖条标题和无竖条标题的标题文字左边缘应保持一致
- 竖条可向标题文字左侧绝对定位偏移展示

### 间距规则
- 表单一级标题与下方表单项间距：8px
- 首个一级标题不额外增加上方间距
- 除首个一级标题外，其他一级标题与上方表单项（Label）间距：16px
- **禁止通过标题文字的 margin-bottom 控制间距**，标题与表单项之间的间距必须由外层布局 gap / margin 控制

## 5.5 常见场景示例

### 5.5.1 基础表单

适用场景：
- 登录信息
- 基础配置
- 简单对象编辑

结构要求：
- 表单采用垂直布局
- 每个表单项包含 Label、输入控件、错误提示区域
- 必填项在 Label 前显示红色星号
- 表单项之间垂直间距为 8px

示例字段：
- 用户名：必填，文本输入框
- 密码：必填，密码输入框

验收标准：
- Label 字号为 12px
- 输入框高度为 32px
- 必填星号显示在 Label 前
- 校验失败时错误提示显示在输入框下方

### 5.5.2 水平表单

适用场景：
- 弹窗表单
- 抽屉表单
- 配置项较多但字段较短的表单

结构要求：
- Label 位于输入控件左侧
- Label 左对齐
- 同一区域内所有 Label 宽度一致
- Label 与输入控件之间间距为 16px
- 输入控件填充剩余空间

示例字段：
- 邮箱：必填，需要符合邮箱格式

验收标准：
- 所有 Label 宽度一致，左对齐
- Label 与输入控件间距为 16px
- 输入控件填充剩余空间

### 5.5.3 禁用表单项

适用场景：
- 系统自动生成字段
- 不允许编辑的配置项
- 依赖其他条件开启的字段

状态要求：
- 输入控件不可编辑
- 使用禁用态背景色（graphite-l30, #E1E5EB）
- 使用禁用态文字颜色（graphite, #A1A7B3）
- 不触发表单校验编辑反馈

验收标准：
- 禁用态背景色与正常态有明显区分
- 禁用态文字颜色与正常态有明显区分
- 禁用态不可编辑
- 禁用态不触发表单校验
