# 标签组件设计与交互规范

## 使用声明

本规范只定义标签组件的视觉样式、语义状态、交互行为和验收标准，不提供具体框架代码实现。

生成代码时，必须以当前项目的技术栈 Skill 为准。

如果本规范与项目技术栈约束发生冲突，优先遵守项目技术栈约束。

本规范禁止包含：
- React / Vue / Angular 代码示例
- JSX / TSX
- TypeScript 示例
- import 语句
- 具体组件库 API
- @ant-design/icons
- @ant-design/icons-vue
- XxxOutlined / XxxFilled / XxxTwoTone 图标组件
- React Hooks
- React.cloneElement
- Vue3 Composition API

本规范中的 Tag、Icon、Dropdown、Input 等名称仅表示设计语义，不代表必须使用某个框架或组件库的同名组件。

所有图标只描述语义，不指定具体图标组件名。

## 标签类型

### 浅色标签

适用场景：
- 次要信息
- 弱化提示
- 分组标识

视觉规范：
- 圆角：2px
- 内边距：4px 8px
- 字号：12px
- 字重：400
- 边框：无
- 文字颜色：根据语义色选择
- 背景颜色：对应文字色的 10% 透明度

验收标准：
- 标签整体高度与 12px 文本和 4px 上下内边距匹配
- 标签背景较轻，不抢主视觉
- 同一语义色的文字与背景必须保持一致

### 深色标签

适用场景：
- 强调信息
- 重要状态
- 品牌展示

视觉规范：
- 圆角：2px
- 内边距：4px 8px
- 字号：12px
- 字重：400
- 边框：无
- 文字颜色：白色 (#FFFFFF)
- 背景颜色：根据语义色选择

验收标准：
- 标签背景与文字对比度足够
- 深色背景下文字清晰可读

### 点状标签

适用场景：
- 状态指示
- 风险等级
- 安全等级
- 简洁分类展示

结构：
- 左侧为圆点
- 右侧为文字

视觉规范：
- 圆点尺寸：6px × 6px
- 圆点圆角：50%
- 圆点与文字间距：4px
- 文字字号：12px
- 文字字重：400
- 文字颜色：graphite-d40 (#2F3540)
- 背景：透明
- 边框：无

安全等级颜色：
- 失陷：#82010E
- 高危：#CF171D
- 中危：#FA721B
- 中低危：#FDAA1D
- 低危：#6F7785
- 信息：#0BA7B5
- 安全：#12A679

验收标准：
- 圆点与文字垂直居中
- 圆点颜色与安全等级一致
- 点状标签不应出现色块背景

### Icon+色块标签

适用场景：
- 任务状态
- 流程状态
- 进度展示

结构：
- 左侧为状态语义图标
- 右侧为状态文字
- 整体使用浅色标签样式

视觉规范：
- 圆角：2px
- 内边距：4px 8px
- 图标尺寸：12px × 12px（使用面性图标）
- 图标与文字间距：4px
- 字号：12px
- 字重：400
- 图标颜色与文字颜色一致
- 背景颜色为文字色 10% 透明度

状态映射：
- 等待中：等待语义图标，蓝色 #1C6EFF
- 进行中：加载语义图标，蓝色 #1C6EFF
- 成功：成功语义图标，绿色 #12A679
- 失败：失败语义图标，红色 #CF171D
- 未完成：等待语义图标，灰色 #6F7785

实现约束：
- 不指定具体图标组件名
- 图标由当前技术栈或组件库映射
- 如果当前组件库没有合适图标，可以省略图标，仅保留文字和色块

验收标准：
- 图标与文字垂直居中
- 图标颜色与文字颜色一致
- 背景为对应颜色的 10% 透明度

### 双色Icon+文字标签

适用场景：
- 品牌标识
- 产品分类
- 强调型标签

结构：
- 左侧为语义图标
- 右侧为文字
- 整体使用双色渐变背景

视觉规范：
- 圆角：2px
- 内边距：4px 8px
- 图标尺寸：14px × 14px（使用面性图标）
- 图标与文字间距：6px
- 字号：12px
- 字重：400
- 文字颜色：白色 (#FFFFFF)
- 图标颜色：白色 (#FFFFFF)
- 背景颜色：从左到右双色渐变

验收标准：
- 渐变方向为从左到右
- 图标和文字在渐变背景上清晰可读
- 渐变色值根据语义选择

### 文字+文字标签

适用场景：
- 分类展示
- 属性标识
- 组合信息展示

结构：
- 文字1 + 分隔符 + 文字2

视觉规范：
- 圆角：2px
- 内边距：4px 8px
- 字号：12px
- 字重：400
- 分隔符：竖线或空格
- 文字间距：4px
- 文字颜色：根据背景选择（深色背景用白色）
- 背景颜色：根据语义选择

验收标准：
- 两段文字视觉上是一个整体
- 分隔符与文字间距一致

### 文字+数字标签

适用场景：
- 安全等级展示
- 风险等级标识
- 威胁级别显示

结构：
- 左侧为文字区
- 右侧为数字区
- 两部分无间距拼接成一个整体标签

视觉规范：
- 整体圆角：2px
- 左侧文字区：深色背景，白色文字
- 右侧数字区：同色系 10% 透明度背景，文字颜色与左侧背景色一致
- 左侧文字区只保留左上、左下圆角
- 右侧数字区只保留右上、右下圆角
- 中间拼接处无圆角
- 两部分之间间距为 0
- 字号：12px
- 字重：400
- 内边距：4px 8px

安全等级定义：
- 未定级：数值 1，颜色 #6F7785
- 安全：数值 2，颜色 #12A679
- 信息：数值 3，颜色 #0BA7B5
- 低危：数值 4，颜色 #6F7785
- 中危：数值 5，颜色 #FA721B
- 高危：数值 6，颜色 #CF171D
- 严重：数值 7，颜色 #82010E

验收标准：
- 左右两段视觉上是一个整体
- 拼接处不能出现间隙
- 拼接处不能出现圆角
- 数字区背景必须是文字区颜色的 10% 透明度

### 可编辑标签

适用场景：
- 标签管理
- 动态添加标签
- 动态删除标签

结构：
- 已有标签列表
- 每个标签右侧有关闭入口
- 标签列表末尾有新增入口
- 新增入口点击后变为输入框

视觉规范：
- 基础样式：灰色浅色标签
- 标签文字颜色：#6F7785
- 标签背景：#F0F1F2
- 标签圆角：2px
- 标签高度：约 28px
- 标签之间间距：4px
- 关闭入口尺寸：12px × 12px
- 关闭入口与文字间距：4px
- 新增入口宽度：20px
- 新增入口高度：28px
- 新增入口白底，边框色 #D3D7DE

交互规则：
1. 点击新增入口后，新增入口切换为输入框
2. 输入框自动获得焦点
3. 输入内容后按 Enter，新增标签
4. 输入框失焦时，如果有有效内容，则新增标签
5. 如果输入为空，不新增标签
6. 如果输入内容已存在，不重复新增
7. 新增完成后清空输入值，并恢复新增入口
8. 点击标签关闭入口时，删除对应标签

验收标准：
- 新增、删除操作不影响其他标签样式
- 标签换行后保持 4px 间距
- 输入框高度与标签高度一致
- 重复标签不会被添加

### 可变更标签

适用场景：
- 状态切换
- 类型选择
- 动态配置

结构：
- 当前状态标签
- 右侧下拉入口
- 下拉选项面板

视觉规范：
- 基础样式：Icon 标签
- 左侧为状态语义图标
- 中间为状态文字
- 右侧为下拉语义图标
- 图标尺寸：12px × 12px
- 图标与文字间距：4px
- 文字字号：12px
- 背景透明
- 点击区域需要可感知

交互规则：
1. 点击标签或下拉入口时，展开选项面板
2. 选项面板展示所有可切换状态
3. 当前状态在选项中需要有选中标识
4. 点击某个选项后，当前标签状态立即更新
5. 状态更新后，下拉面板关闭
6. 状态变化需要同步更新文字、颜色和图标语义
7. 如果状态切换会影响业务数据，需要触发外部变更事件

验收标准：
- 点击后能展开选项
- 选择后能更新当前状态
- 当前状态颜色与状态映射一致
- 下拉面板关闭后页面状态保持正确

## 标签文本折行与截断规范

为了防止因标签文本过长导致页面排版错位（尤其是在表格、列表等紧凑布局中），必须严格遵守以下规范：

1. **绝对禁止换行**：标签（Tag）仅允许单行展示，**严禁在标签内发生任何折行行为**。
2. **防挤压防折行实现**：在 CSS/Tailwind 布局中，必须对标签元素（或其父级 Flex 项）设置 `flex-shrink-0` 和 `whitespace-nowrap`，以防空间受限时标签被强行挤压变形或换行。
3. **超长截断（Ellipsis）**：若标签文案确实较长且空间受限，文字必须进行单行截断并以省略号 `...` 结尾。
   * **实现方式**：设置最大宽度（例如：`max-w-[80px]` 或根据业务微调）并添加截断类名 `truncate`（或 `overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`）。
4. **悬浮显示全称**：凡是进行了截断的标签，**必须绑定鼠标悬浮提示**（例如：设置 HTML 原生 `title` 属性或使用 Tooltip 组件），保证用户鼠标悬浮在标签上时能看到完整的文字。
