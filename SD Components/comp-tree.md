# 树组件 (Tree)

## 使用声明

本规范只定义组件的视觉结构、布局关系、交互行为和验收标准，不提供具体框架代码实现。

生成代码时，必须以当前项目的技术栈 Skill 为准。

如果本规范与项目技术栈约束发生冲突，优先遵守项目技术栈约束。

本规范禁止包含：
- React / Vue / Angular 代码示例
- JSX / TSX
- TypeScript 示例
- import 语句
- className
- React state 写法
- 具体图标组件名
- 具体组件库 API

本规范中的 Tree 等名称仅表示设计语义，不代表必须使用某个框架或组件库的同名组件。

# 第一部分：树组件规范

## 一、视觉规范

### 默认状态
- 字号：12px
- 字重：400
- 文字颜色：var(--color-graphite-d40)
- 节点图标颜色：var(--color-graphite-d20)
- 展开/收起图标颜色：var(--color-graphite-d20)

### 选中状态 (Selected)
- 节点背景色：var(--color-blue-l50)
- 节点文字颜色：var(--color-blue)

### 悬浮状态 (Hover)
- 节点背景色：var(--color-graphite-l50)

### 拖拽状态
- 拖拽插入位置上方：顶部 2px 蓝色线条，颜色 var(--color-blue)
- 拖拽插入位置下方：底部 2px 蓝色线条，颜色 var(--color-blue)
- 拖拽插入位置内部：背景色 var(--color-blue-l50)

## 二、结构规范

### 节点内容区
- 节点高度：32px
- 节点图标与文字间距：4px
- 层级缩进：每级 16px

### 树组件无图标模式规范

树组件支持两种模式：

1. 有业务图标模式：节点包含展开图标、业务图标、文字
2. 无业务图标模式：节点只包含展开图标、文字，不显示业务图标，也不预留业务图标占位

**核心原则：当树为无业务图标模式时，不得预留图标占位；层级缩进和连接线必须以节点文字起始区域为对齐基准，下一层级的展开/收起图标左边缘应与上一层级文字起始区域对齐。**

#### 无图标模式结构

```text
[展开/收起图标][文字]
```

不得渲染：文件夹图标、文件图标、业务图标占位、空的 icon 容器

#### 对齐规则

无图标模式下，下一层级的展开/收起图标，应与上一层级的文字起始区域对齐：

```text
第 1 级：[展开图标] 文字
第 2 级：          [展开图标] 文字
```

第 2 级展开图标的左边缘，应与第 1 级文字起始区域对齐。

#### 连接线规则

无图标模式下，连接线不再对齐业务图标轨道，而是对齐文字轨道：

- 连接线必须跟随文字起始区域展开
- 子级连接线必须能看出属于上一级文字节点
- 不得因为缺少业务图标而保留空 icon 位
- 不得让连接线仍然对齐原来的文件夹图标位置
- 不得出现"文字在一列、连接线在另一列"的错位效果

#### 与有图标模式的区别

有图标模式：

```text
第 1 级：[展开图标][业务图标][文字]
第 2 级：          [展开图标][业务图标][文字]
```

下一层级展开图标对齐上一层级业务图标。

无图标模式：

```text
第 1 级：[展开图标][文字]
第 2 级：        [展开图标][文字]
```

下一层级展开图标对齐上一层级文字起始区域。

#### 无图标模式禁止项

- 禁止无图标模式仍然预留业务图标宽度
- 禁止渲染空的 icon 占位元素
- 禁止连接线按"有图标树"的轨道计算
- 禁止让子级节点仅靠缩进表达层级，连接线必须跟随文字区域
- 禁止无图标树中出现文件夹、文件、默认 Ant Design 图标

### 节点 padding 拆分定义

节点 padding 必须拆分定义，不得笼统使用"节点内边距"：

- Tree 面板外层 padding：与搜索工具栏一致，左右 12px
- Ant Tree treenode：padding 必须清零
- switcher 容器：padding / margin 必须清零
- content wrapper：仅允许右侧保留 8px padding，用于内容与右侧操作区留白
- title slot 根容器：不额外设置左 padding
- hover / selected 背景：跟随 content wrapper 宽度铺开，不得覆盖 switcher 和连接线

### 统计数字
- 统计数字跟随树文字一起显示，紧跟在文字右侧
- 统计数字颜色：var(--color-graphite)
- 统计数字字号：12px
- 统计数字与文字间距：4px
- 禁止将统计数字放在右侧（影响操作图标显示）

### 树节点 Hover 操作区规范

#### 结构规则

树节点内容必须拆分为左右两部分：

左侧内容区：业务图标 + 节点名称 + 统计数字
右侧操作区：更多操作 icon

- "更多"操作 icon 不得作为普通文本后缀跟在节点名称后面，必须作为节点行右侧的独立操作区
- 节点 title 容器必须撑满整行可用宽度
- 左侧内容区自适应宽度
- 右侧操作区固定在节点行最右侧
- 统计数字仍然紧跟节点名称显示，不得为了放置操作 icon 而右对齐
- hover 显示操作 icon 时，节点文字、统计数字不得发生位移或抖动

#### title slot flex 布局结构

title slot 根容器必须使用横向 flex 布局，并拆成左右两块：

- 左侧内容区：业务图标 + 节点名称 + 统计数字
- 右侧操作区：更多操作按钮
- 左侧内容区必须 min-width: 0，允许文字省略
- 右侧操作区必须 margin-left: auto，固定在节点行最右侧
- 更多操作按钮的外层点击热区不得小于 24px × 24px
- 更多 icon 本体可以是 16px × 16px，但不可作为唯一点击区域
- 禁止把更多 icon 作为节点名称后的 inline 后缀

#### 点击热区

- 更多 icon 本体可为 16px × 16px
- 外层点击热区不得小于 24px × 24px
- 不得只让 SVG 图标本身作为点击区域
- 操作区需要保证鼠标容易悬停和点击

#### 显示规则

- 默认状态隐藏更多操作 icon
- hover 当前节点时显示该节点的更多操作 icon
- 当前节点选中时，可保持更多操作 icon 可见
- 只显示当前 hover / selected 节点的操作 icon，不得多个节点同时显示
- 操作图标：横向三点图标
- 图标颜色：var(--color-graphite-d20)
- 图标 hover 背景色：var(--color-graphite-l50)
- 图标内边距：4px
- 图标圆角：2px

#### 点击交互

点击更多操作 icon 时：
- 弹出操作面板
- 不触发节点选中
- 不触发展开 / 收起
- 不触发拖拽
- 操作面板必须以更多 icon 的点击热区为锚点弹出
- 必须处理事件冒泡，确保点击更多 icon 只执行打开操作面板

#### 操作面板

- 操作面板从更多 icon 附近弹出
- 建议与更多 icon 右侧或右上角对齐
- 面板不得遮挡当前节点文字
- 面板层级必须高于树节点
- 面板关闭后，不得造成 hover / selected 状态异常残留

#### 禁止项

- 禁止把更多 icon 直接放在节点文字后面
- 禁止让 SVG 本身成为唯一点击区域
- 禁止用 margin-left 强行把 icon 推到右侧
- 禁止把统计数字右对齐来给操作 icon 让位
- 禁止点击操作 icon 时同时触发节点选中、展开、收起或拖拽
- 禁止将操作面板绑定到整行节点上

## 三、交互规则

### 搜索过滤
- 输入关键词自动过滤匹配节点，不匹配的节点隐藏
- 匹配节点的所有父节点自动展开
- 清空搜索后恢复搜索前的展开状态

### 节点选择
- 点击节点触发选中
- 选中态必须有清晰视觉反馈
- 支持单选和多选模式

### 默认展开规则
- 默认展开根目录，显示根目录下一级内容

### 节点展开/收起
- 点击展开/收起图标切换子节点可见性
- 展开动画过渡自然

### 拖拽排序
- 支持拖拽节点调整位置
- 拖拽时需显示插入位置指示器
- 支持插入目标节点的前、后、内部三种位置

## 四、虚拟滚动
- 超长树结构必须开启虚拟滚动
- 需要明确设置容器高度

## 五、Ant Design Vue Tree 实现约束

组件底座：Ant Design Vue Tree

必须启用：
- showLine：显示树连接线
- blockNode：节点整行可选中
- showIcon：**必须关闭，不得启用**
- switcherIcon：自定义方形展开/收起图标
- draggable：需要拖拽时开启
- height：超长树开启虚拟滚动
- selectedKeys：受控选中节点
- expandedKeys：受控展开节点

### 禁止使用默认图标体系

**总则：禁止使用 Ant Design Vue Tree 默认图标体系**（含默认三角展开图标、默认 file icon、默认小尺寸图标）。

具体要求：
- 业务图标必须统一在 title slot 内自定义渲染，不得使用默认 title 字符串
- showLine 仅用于连接线，不得产生默认 leaf file icon
- 同一节点左侧只能出现一个业务图标
- 展开/收起方形 icon 属于 switcher，不算业务图标

节点内容自定义包含：节点图标、节点名称、统计数字、hover 后显示的行内操作按钮。

原因：规范中有"统计数字紧跟文字右侧""hover 显示三点操作""选中后图标变蓝"等要求，这些都需要自定义 title slot 才能稳定实现。

### 图标尺寸与层级对齐规范

分组树内所有可见图标尺寸固定为 **16px × 16px**（含展开图标、收起图标、业务文件夹图标、叶子节点业务图标）。

### 方形展开 / 收起图标规则

必须通过 switcherIcon 自定义为方形按钮，展开为减号，收起为加号。

- 图标尺寸：16px × 16px
- 背景色：浅色底
- icon 颜色：var(--color-graphite-d20)

switcher 容器：
- 容器宽度：20px
- 容器高度：32px
- 内图标水平、垂直居中
- 不得让 16px 图标直接影响 32px 行高
- switcher 内部包裹元素和 SVG 本身也必须脱离 inline baseline，使用 flex 在 32px 行高中垂直居中

第一层级根节点零左边距：
- 根节点零左边距指的是 Tree 内部根节点不保留 Ant Design Vue 默认缩进，不是指 Tree 区域贴到面板最左侧
- 树区域外层必须与搜索工具栏使用同一条左侧视觉基准线
- 例如搜索工具栏左侧 icon 按钮距离面板左侧为 12px，则 Tree 区域也应保留 12px 外层 padding；在该 padding 内部，第一层级根节点不得再额外增加 Ant Tree 默认缩进

正确关系：
```
面板左边缘 → 12px 面板内容内边距 → 工具栏 icon 按钮左边缘
面板左边缘 → 12px 面板内容内边距 → 第一层级根节点 switcher 左边缘
```

- `.ant-tree-switcher` 不得保留任何 padding 或 margin

### 32px 行高对齐规则

当分组树节点高度设为 32px 时，必须同时覆盖 Ant Tree 外层节点、switcher、content wrapper 和 title slot 容器高度。

不得只设置 title slot 的高度。

所有节点相关容器必须统一为 32px 高度，并使用 flex 垂直居中。

验收要求：
- 方形展开/收起 icon 必须位于 32px 行高的垂直中心
- 文件夹业务 icon 必须位于 32px 行高的垂直中心
- 节点文字必须位于 32px 行高的垂直中心
- 不允许 switcher、业务 icon、文字三者出现上下错位

### 分组树连接线规则

分组树连接线用于表达父子层级关系，连接线为实线。

#### 一、连接线实现必须二选一，不得混用

方案 A（默认）：使用 Ant Design Vue `showLine` 作为唯一连接线来源。
- CSS 只允许调整 AntD 原有连接线的颜色、位置、宽度
- 不得新增 `switcher::before` / `switcher::after` / `noop::after` 横线

方案 B：完全关闭 `showLine`，自行实现整套连接线。
- 必须自己处理 ancestor 竖线、当前节点横线、最后一个子节点截断、叶子节点横线

**默认必须使用方案 A。** 除非 AntD `showLine` 无法满足设计，并且明确选择方案 B，否则禁止混用两套连接线。

#### 二、多层级连接要求

连接线必须完整表达父子关系，不得只显示到第二级。

对于三级及更深层级，必须显示：

- 父级竖向连接线
- 子节点横向连接短线
- 同级子节点之间的竖向延续线
- 最后一个子节点处的结束线

叶子节点也必须显示连接线，不得因为没有展开/收起按钮而隐藏连接线。

父级竖线规则：
- 父级竖线必须由层级缩进区域承载，而不是只画在当前节点 switcher 上
- 每一层 ancestor indent 都必须判断是否还有后续兄弟节点
- 只允许在当前层级的最后一个节点处结束竖线
- 不得因为某个展开节点的子树结束，就截断更上一级父节点的竖线

正确示意：

```text
全部设备
└─ 广州办事处
   ├─ 办公区
   └─ 机房
```

禁止项：
- 禁止第三级节点只有缩进，没有连接线
- 禁止叶子节点丢失横向连接短线
- 禁止父节点与子节点之间断线
- 禁止只依赖文字缩进表达父子关系
- 禁止出现"办公区/机房看不出属于广州办事处"的效果

#### 三、叶子节点规则

叶子节点没有展开/收起按钮，但仍必须保留 switcher 占位并参与连接线展示：

- 不显示加号/减号
- 不显示默认 file icon
- 必须保留 switcher 占位
- 必须显示一根横向连接短线（只能有一根）
- 不得因隐藏 leaf icon 而隐藏连接线

### 搜索过滤逻辑

1. 搜索前保存 expandedKeys 快照
2. 输入关键词后递归过滤 treeData
3. 保留命中节点及其父节点
4. 自动展开所有命中节点的父节点
5. 清空搜索后恢复原始 treeData
6. 清空搜索关键词后，不得将 expandedKeys 置空，必须将 expandedKeys 恢复为搜索前保存的快照
7. 搜索前快照只在从"非搜索状态"进入"搜索状态"的第一次输入时保存；搜索过程中继续输入关键词，不得反复覆盖该快照

## 六、AI 编程执行约束

当根据本规范开发分组树组件时，必须先完成规范映射，再开始编码。

### 一、开发前必须确认

在编码前必须明确以下实现决策：

1. 使用 Ant Design Vue Tree 作为结构底座
2. 禁用 Ant Design Vue 默认节点图标体系
3. 业务图标只允许在 title slot 中自定义渲染
4. 展开/收起图标只允许通过 switcherIcon 自定义渲染
5. 连接线默认使用 AntD showLine 作为唯一来源（方案 A）
6. 节点高度统一为 32px
7. Tree 外层左侧基准线必须与搜索工具栏左侧 icon 按钮对齐
8. hover 更多操作按钮必须是独立右侧操作区，不得作为文字后缀

### 二、不得直接使用 AntD 默认效果

不得直接依赖以下 Ant Design Vue Tree 默认行为：

- 默认三角展开图标
- 默认 PlusSquare / MinusSquare 图标
- 默认 file icon
- 默认 folder icon
- 默认 title 字符串渲染
- 默认节点高度
- 默认 content wrapper padding
- 默认 switcher padding
- 默认 selected / hover 样式
- 默认缩进宽度

凡是影响视觉还原的 AntD 默认样式，都必须显式覆盖。

### 三、修改已有代码时必须先清理旧实现

如果已有代码中存在以下内容，必须先删除或改造：

- `showIcon` 属性开启（必须关闭）
- `treeIcon` / `icon` 属性使用默认图标
- 默认 `title` 字符串渲染（必须改为 title slot 自定义渲染）
- 默认三角展开图标（必须改为 switcherIcon 方形加减号）
- 自行添加的 `switcher::before` / `switcher::after` / `noop::after` 伪元素连接线（与 showLine 冲突时必须移除）
- 笼统的"节点内边距"设置（必须按本规范拆分定义）
- 更多 icon 作为文字后缀 inline 放置（必须改为 flex 右侧操作区）

## 七、验收标准
- 节点高度一致，不因内容变化抖动
- 选中态背景色和文字颜色正确
- 悬浮态背景色正确
- 搜索过滤后展开状态正确
- 清空搜索后恢复原始展开状态
- 拖拽插入位置指示清晰
- 统计数字显示在文字右侧，不与操作图标冲突
- hover 节点时，更多操作 icon 固定显示在该行最右侧
- 更多操作 icon 有独立点击热区（不小于 24px × 24px），点击稳定
- 点击更多 icon 能弹出操作面板，不影响节点选中、展开、收起
- hover 显示操作 icon 时，文字和统计数字不发生位移
- Tree 外层左侧基准线与搜索工具栏左侧 icon 按钮对齐
- 根节点不保留 AntD 默认缩进，但不贴面板最左侧
- 连接线不出现双横线或断线

# 第二部分：应用场景

## 一、左树分组组件

分组树是由"顶部标题栏 + 搜索工具栏 + Tree 区域"组成的复合组件，不是单独的 Tree 组件，必须包含完整面板结构：

```
分组树面板
├─ 顶部标题栏
│  ├─ 标题：分组

```

## 二、左树统计型平铺树规范

### 组件定位

统计型平铺树用于展示一组分类筛选项及其统计数量。

该组件虽然位于树面板区域，但不表达父子层级关系，不显示树连接线，不显示展开/收起图标，也不显示业务图标。

适用场景：

```text
全部（100）
类别一（40）
类别二（30）
类别三（10）
类别四（10）
类别五（10）
```

### 数据结构规则

统计型平铺树为单层结构：

1. 顶部标题栏：跟左树分组组件一致
2. 搜索工具栏：跟左树分组组件一致，但只有搜索框
3. 树组件区域：
   - 第一项固定为"全部"
   - "全部"表示当前统计范围内的总数
   - "全部"不是父节点，不可展开
   - "全部"下方的类别项不是"全部"的子节点
   - 所有分类项与"全部"属于同一组筛选项
   - 不得渲染成多级树结构

错误理解：

```text
全部
├─ 类别一
├─ 类别二
└─ 类别三
```

正确理解：

```text
全部（100）
类别一（40）
类别二（30）
类别三（10）
类别四（10）
类别五（10）
```

### 视觉规则

#### 行结构

每一行由以下内容组成：分类名称 + 统计数量 + 右侧操作区

- 分类名称显示在左侧
- 统计数量紧跟分类名称右侧，不右对齐
- 操作按钮固定在当前行最右侧，不占用文字区域

#### 图标规则

统计型平铺树不显示任何节点图标。禁止出现：

- 展开/收起图标
- 文件夹图标
- 文件图标
- 默认 Ant Design Tree 图标
- 空的 icon 占位
- 树连接线

#### "全部"项规则

"全部"是统计型平铺树的固定首项：

- 固定显示在分类列表第一行
- 文案为"全部"
- 数量显示为总数，例如（100）
- 不显示更多操作按钮
- 不显示 hover 操作区
- 不显示展开/收起状态
- 不参与层级缩进
- 点击后切换为全部筛选状态

#### 分类项规则

分类项用于切换具体分类筛选：

- 分类名称与统计数量同行显示
- 统计数量格式为中文括号或半角括号，按项目统一，例如 类别一（40）
- 选中项文字颜色为蓝色
- 选中项统计数字颜色跟随文字变为蓝色
- 未选中项使用默认文字色
- hover 时显示右侧更多操作按钮
- 点击分类项切换当前筛选条件

#### 选中态规则

选中分类项时：

- 分类名称变为 var(--color-blue)
- 统计数量同步变为 var(--color-blue)
- 字重可提升
- 不显示树节点背景，除非业务规范明确要求
- 不显示展开状态
- 不显示左侧指示线或圆点

示例：

```text
全部（100）
类别一（40）   ← 选中，文字和数字为蓝色
类别二（30）
```

#### 行内操作按钮规则

分类项 hover 时，右侧显示更多操作按钮：

- 更多操作按钮固定在当前行右侧
- 不得贴在文字或统计数字后面
- 操作按钮必须有独立点击热区（建议 24px × 24px 或 28px × 28px）
- 点击更多操作按钮时弹出操作面板
- 点击更多操作按钮不得触发分类选中
- 点击更多操作按钮不得影响当前筛选状态

### 布局规则

统计型平铺树不使用层级缩进：

- 所有行左侧文字起点一致
- "全部"和所有分类项左对齐
- 不因缺少图标而预留图标空位
- 不因使用树组件而产生默认 indent
- 不显示连接线区域
- 不显示 switcher 占位区域

### Ant Design Vue 实现约束

统计型平铺树不建议使用 a-tree 实现。

推荐使用自定义列表结构：

```
统计型平铺树容器
├─ 全部项
├─ 分类项
├─ 分类项
└─ 分类项
```

如果必须使用 Ant Design Vue Tree，也必须关闭树形能力：

- 不启用 showLine
- 不启用 showIcon
- 不渲染 switcherIcon
- 不保留 indent 占位
- 不保留 leaf icon 占位
- 不显示展开/收起状态
- 不得出现树连接线

### 禁止项

- 禁止把"类别一/类别二/类别三"渲染成"全部"的子节点
- 禁止显示树连接线
- 禁止显示展开/收起图标
- 禁止显示文件夹或文件图标
- 禁止保留图标占位
- 禁止分类项产生层级缩进
- 禁止更多操作按钮贴在文字后方
- 禁止点击更多操作按钮时触发当前行选中
