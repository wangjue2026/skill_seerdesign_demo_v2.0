# 详情概览卡片 / Detail Overview Card

## 适用场景

用于详情页、详情抽屉、详情弹窗顶部，展示当前对象的核心身份信息、关键属性和主操作。

常见场景：

- 设备详情
- 任务详情
- 用户详情
- 策略详情
- 升级包详情
- 规则详情

## 结构

详情概览卡片从左到右分为三块：

```
[对象图标] [核心信息区] [右侧操作区]
```

### 结构说明

| 区域 | 说明 |
|------|------|
| 对象图标 | 用于强化对象类型，例如设备、任务、用户、策略 |
| 核心信息区 | 展示名称、状态、关键属性 |
| 右侧操作区 | 展示主操作 / 次操作 / 更多操作 |

## 卡片容器规范

| 属性 | 规范值 |
|------|--------|
| 宽度 | 100% |
| 背景色 | var(--color-blue-l50) |
| 圆角 | 2px |
| 内边距 | 四周 16px |
| 高度 | 根据内容自适应 |

- 卡片不额外突破页面、抽屉、弹窗内容区边距
- 如果在抽屉中使用，卡片应跟随 ant-drawer-body 的内容区宽度，不额外增加左右外边距

## 对象图标区

| 属性 | 规范值 |
|------|--------|
| 图标尺寸 | 32px × 32px |
| 图标颜色 | var(--color-blue) |
| 图标区宽度 | 32px |
| 图标区不压缩 | 是 |
| 图标与核心信息区间距 | 24px |
| 图标顶部与核心信息区顶部对齐 | 是 |

### 大图标概览

若单独说明要做大图标概览，则显示 64px × 64px 的图标背景块，里面放置符合对象语意的图标。

## 核心信息区

核心信息区包含：

- 第一行：对象名称 + 状态
- 第二行及之后：关键属性信息

### 对象名称

| 属性 | 规范值 |
|------|--------|
| 字号 | 14px |
| 字重 | 600 |
| 颜色 | var(--color-graphite-d40) |
| 行高 | 22px |

示例：gz-device-1

### 状态标签

- 状态跟随对象名称右侧展示
- 状态与名称间距：8px
- 状态高度：20px 或 Tag 规范
- 状态字号：12px
- 状态颜色遵循状态标签规范

示例：gz-device-1 [在线]

### 关键属性

属性以 Label + Value 形式横向排列。

| 属性 | 规范值 |
|------|--------|
| 字号 | 12px |
| 行高 | 20px |
| Label 颜色 | var(--color-graphite) |
| Value 颜色 | var(--color-graphite-d40) |
| 属性项之间横向间距 | 最小 16px，根据横向宽度自适应，最大 32px |
| 多行属性之间行间距 | 8px |
| Label 与 Value 间距 | 16px |

排列规则：

- 多行属性时每列 Label 左对齐，内容也左对齐
- 多行属性时按每列最长 Label 计算与 Value 的间距
- Value 较长时用省略号收尾，不允许自行换行

示例：

```
设备类型  AF          所属分组  广州办事处      地理位置  广东省/广州市/番禺区
```

结构上每个属性为独立片段：

```
[设备类型 AF]   [所属分组 广州办事处]   [地理位置 广东省/广州市/番禺区]
```

## 右侧操作区

右侧操作区用于放按钮组，例如：进入设备、编辑、更多。

| 属性 | 规范值 |
|------|--------|
| 位置 | 固定在卡片右侧 |
| 对齐方式 | 按钮组顶部对齐 |
| 按钮之间间距 | 12px |
| 按钮高度 | 32px |
| 按钮字号 | 12px |
| 右侧操作区不压缩 | 是 |
| 与核心信息区最小间距 | 8px |

重点规则：

- 无论核心信息区有几行，右侧按钮组始终位于卡片右上角
- 按钮不随卡片高度增加而垂直居中

## 布局规则

1. 卡片采用横向布局
2. 左侧图标区固定 32px，不压缩
3. 中间核心信息区自适应，占据剩余空间
4. 右侧操作区固定在右侧，不压缩
5. 核心信息较多时允许换行，卡片高度自适应增加
6. 右侧按钮组始终顶部对齐，不垂直居中
7. 核心信息不得与右侧按钮重叠
8. 卡片不得超出页面、抽屉、弹窗内容区
