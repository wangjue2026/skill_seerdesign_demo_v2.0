# 弹窗组件设计规范

## ⚠️ 高频违规速查（阅读正文前必看）

> AI 在生成弹窗代码前，必须先逐行核对此表，再阅读下方完整规范。

| 规则 | ✅ 正确 | ❌ 常见错误 |
|------|--------|------------|
| **底部按钮对齐方向** | **右对齐**，主按钮在左、取消按钮在右（整组按钮右对齐） | 左对齐（与抽屉相反，极易搞混） |
| 主按钮颜色 | `var(--color-blue)` — **任何场景的主按钮都是蓝色** | 用红色作为危险操作的主按钮 |
| 宽度可选范围 | `400 / 480 / 640 / 720 / 800 / 960 / 1200px` | 使用规范之外的任意像素值 |
| 标题栏高度 | 固定 `48px` | 高度未锁定 |
| 底部按钮栏高度 | 固定 `56px` | 高度未锁定或使用 `48px` |
| 标题栏底部分割线 | **无分割线** | 加了 `border-bottom` |
| 底部栏顶部分割线 | **无分割线** | 加了 `border-top` |
| 内容区左右边距 | `32px` | 使用 `16px` 或 `24px` |
| 底部按钮字号 | `14px` | 沿用默认 `12px` |
| 按钮间距 | `8px` | 使用 `12px` 或 `16px` |
| 内容区溢出处理 | 仅内容区滚动，标题栏和底部栏**固定不动** | 整个弹窗一起滚动 |

## 使用声明

本规范只定义弹窗的视觉、布局、交互和验收规则，不提供具体框架代码实现。

生成代码时，必须以当前项目的技术栈 Skill 为准。

如果本规范与项目技术栈约束发生冲突，优先遵守项目技术栈约束。

禁止在本规范中包含：
- React / Vue / Angular 代码示例
- JSX / TSX
- TypeScript 示例
- import 语句
- 具体组件库 API
- 具体组件库内部 class 的全局覆盖
- open / styles / footer 等特定版本 API 示例

本规范中的 Modal、Footer、Button 等名称仅表示设计语义，不代表必须使用某个框架或组件库的同名 API。

## 核心规范

- 标题栏高度：48px
- 底部按钮栏高度：56px
- 按钮区域：左对齐，主按钮左间距 16px
- 按钮高度：32px
- 按钮文字：14px
- 按钮之间间距：8px
- 内容区域左右边距：32px
- 内容超出时，在内容区域内部滚动
- 弹窗宽度只能从以下尺寸中选择（根据实际 Label 数量和组件宽度选择）：
  - 400px — 纯提示弹窗
  - 480px — 纯提示弹窗，或仅有两三个简单配置项
  - 640px — 两三个简单配置项
  - 720px — 复杂表单配置，需确保表单组件清晰不拥挤
  - 800px
  - 960px
  - 1200px

## 结构

弹窗从上到下分为三层：

1. 标题栏
2. 内容区域
3. 底部按钮栏

## 标题栏

- 高度固定为 48px
- 标题文字垂直居中
- 标题文字左边距 16px
- 标题文字字号 16px，字重 600
- 无底部分割线

## 内容区域

- 左右边距为 32px
- 上下无额外间距，内容区域紧贴标题栏底部和底部按钮栏顶部
- 上下不得增加额外的 padding 或 margin
- 内容超出时，内容区域内部滚动
- 表单、表格、说明信息均不得超出内容区域
- 内容区域不能撑开弹窗整体高度导致底部按钮栏不可见

## 底部按钮栏

- 高度固定为 56px
- 无顶部分割线
- 按钮右对齐
- 按钮之间间距为 8px
- 排列顺序：主按钮在左，取消按钮在右（按钮组整体右对齐）

## 验收标准

- 标题栏高度稳定为 48px
- 底部按钮栏高度稳定为 56px
- 内容溢出时只在内容区域滚动
- 底部按钮栏始终可见
- 按钮右对齐
- 弹窗宽度符合允许尺寸列表
