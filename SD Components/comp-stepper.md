# 步骤条 Stepper 规范

## 1. 组件定位

步骤条用于表达任务流程的步骤总数、当前步骤、步骤状态与完成进度。

适用场景：
1. 多步骤表单
2. 向导式配置流程
3. 审批流 / 流程流转
4. 页面级任务流程

使用原则：
- 步骤数量不少于 2 步。
- 建议不超过 7 步。
- 步骤过多或流程复杂时，应优先拆分流程。

---

## 2. 横向步骤条

横向步骤条是默认形态，适用于页面顶部流程指引、配置向导、表单步骤等场景。

### 2.1 结构

```
横向步骤条
├─ 步骤原子
│  ├─ 步骤图标 / 序号
│  ├─ 步骤标题
│  └─ 步骤描述（可选）
├─ 连接线
└─ 下一个步骤原子
```

### 2.2 布局规则

- 步骤从左到右排列。
- 步骤图标位于标题左侧。
- 标题与描述位于图标右侧。
- 描述位于标题下方。
- 步骤之间通过水平虚线连接。

### 2.3 尺寸与间距

| 项目 | 规范 |
|------|------|
| 步骤图标尺寸 | 24px × 24px |
| 图标与标题间距 | 8px |
| 标题与描述间距 | 4px |
| 步骤之间最小间距 | 80px |
| 连接线样式 | 1px 虚线 |
| 连接线位置 | 与步骤图标圆心水平对齐 |
| 步骤标题字号 | 14px |
| 步骤描述字号 | 12px |

### 2.4 使用规则

- 横向步骤条中，标题优先放在步骤图标右侧。
- 当展示空间不足时，可适当减少描述信息。
- 横向步骤条不建议承载过长标题。

---

## 3. 纵向步骤条

纵向步骤条用于步骤较多、横向空间不足，或需要左侧展示步骤、右侧展示内容的场景。

### 3.1 结构

```
纵向步骤条
├─ 步骤原子
│  ├─ 步骤图标 / 序号
│  ├─ 步骤标题
│  └─ 步骤描述（可选）
├─ 垂直连接线
└─ 下一个步骤原子
```

### 3.2 布局规则

- 步骤从上到下排列。
- 步骤图标位于每个步骤左侧。
- 标题位于图标右侧。
- 描述位于标题下方。
- 步骤之间通过垂直虚线连接。
- 右侧可展示当前步骤对应的业务内容。

### 3.3 尺寸与间距

| 项目 | 规范 |
|------|------|
| 步骤图标尺寸 | 24px × 24px |
| 图标与标题间距 | 12px |
| 标题与描述间距 | 4px |
| 步骤之间垂直间距 | 48px |
| 垂直连接线样式 | 1px 虚线 |
| 连接线位置 | 与步骤图标圆心垂直对齐 |
| 步骤标题字号 | 14px |
| 步骤描述字号 | 12px |

### 3.4 使用规则

- 纵向步骤条适合展示更多步骤信息。
- 纵向步骤条可与右侧内容区组成左右布局。
- 页面操作按钮应放在右侧内容区底部，不应放在步骤条内部。

### 3.5 间距与连接线规则

- 纵向步骤条的步骤间距必须包含在步骤项内部高度中，不得通过外部 margin/gap 实现
- 连接线必须覆盖两个步骤图标之间的完整垂直空间

---

## 4. 步骤原子样式

每一个步骤都由一个"步骤原子"组成。

```
步骤原子
├─ 图标区
│  └─ 序号 / 对勾 / 感叹号
└─ 文案区
   ├─ 步骤标题
   └─ 步骤描述（可选）
```

通用规则：
- 图标区固定 24px × 24px。
- 标题为必填。
- 描述为可选。
- 标题应简短明确，建议 2-8 个字。
- 可选步骤需要在标题中标记"选填"。
- 描述只用于补充说明，不承载核心操作信息。

---

## 5. 步骤状态规范

步骤状态统一分为四类：

1. 进行中
2. 待进行
3. 已完成
4. 错误

### 5.1 进行中

用于表示当前正在处理的步骤。

| 元素 | 规范 |
|------|------|
| 图标样式 | 蓝色实心圆 |
| 图标尺寸 | 24px × 24px |
| 图标内容 | 当前步骤序号 |
| 图标背景 | var(--color-blue) |
| 序号颜色 | #FFFFFF |
| 标题字号 | 14px |
| 标题字重 | 600 |
| 标题颜色 | var(--color-graphite-d40) |
| 描述字号 | 12px |
| 描述字重 | 400 |
| 描述颜色 | var(--color-graphite) |
| 前置连接线 | 蓝色虚线，颜色 var(--color-blue) |
| 后置连接线 | 灰色虚线，颜色 var(--color-graphite-l20)（二级分割线色） |

结构关系：
```
[蓝色序号圆]  标题
              描述文本
```

### 5.2 待进行

用于表示尚未开始的步骤。

| 元素 | 规范 |
|------|------|
| 图标样式 | 灰色描边圆 |
| 图标尺寸 | 24px × 24px |
| 图标内容 | 步骤序号 |
| 图标背景 | 透明 |
| 图标边框 | 1px solid var(--color-graphite-l30) |
| 序号颜色 | var(--color-graphite-l20) |
| 标题字号 | 14px |
| 标题字重 | 400 |
| 标题颜色 | var(--color-graphite-l10) |
| 描述字号 | 12px |
| 描述字重 | 400 |
| 描述颜色 | var(--color-graphite-l10) |
| 连接线 | 灰色虚线，颜色 var(--color-graphite-l20)（二级分割线色） |

结构关系：
```
[灰色序号圆]  标题
              描述文本
```

### 5.3 已完成

用于表示已经完成的步骤。

| 元素 | 规范 |
|------|------|
| 图标样式 | 蓝色描边圆 |
| 图标尺寸 | 24px × 24px |
| 图标内容 | 对勾 |
| 图标背景 | 透明 |
| 图标边框 | 1px solid var(--color-blue) |
| 对勾颜色 | var(--color-blue) |
| 标题字号 | 14px |
| 标题字重 | 400 |
| 标题颜色 | var(--color-graphite-d40) |
| 描述字号 | 12px |
| 描述字重 | 400 |
| 描述颜色 | var(--color-graphite) |
| 后置连接线 | 蓝色虚线，颜色 var(--color-blue) |

结构关系：
```
[蓝色对勾圆]  标题
              描述文本
```

### 5.4 错误

用于表示当前步骤或流程节点异常。

| 元素 | 规范 |
|------|------|
| 图标样式 | 红色描边圆 |
| 图标尺寸 | 24px × 24px |
| 图标内容 | 感叹号 |
| 图标背景 | 透明 |
| 图标边框 | 1px solid var(--color-red) |
| 感叹号颜色 | var(--color-red) |
| 标题字号 | 14px |
| 标题字重 | 400 或 600 |
| 标题颜色 | var(--color-graphite-d40) 或 var(--color-red) |
| 描述字号 | 12px |
| 描述字重 | 400 |
| 描述颜色 | var(--color-graphite) 或 var(--color-red) |
| 连接线 | 默认灰色虚线，颜色 var(--color-graphite-l20)（二级分割线色），除非业务要求错误路径高亮 |

结构关系：
```
[红色感叹号圆]  标题
                描述文本
```

## 6. Ant Design Vue 实现约束

使用 Ant Design Vue Steps 作为底座。

实现要求：
1. 横向步骤条使用 horizontal 方向。
2. 纵向步骤条使用 vertical 方向。
3. 使用 current 控制当前步骤。
4. 使用 status 表达进行中、待进行、已完成、错误状态。
5. 默认样式不满足规范时，需要覆盖字号、字重、颜色、图标尺寸、连接线样式和间距。
6. 不建议重新造一套步骤条结构，除非 Ant Design Vue Steps 无法满足业务状态。

硬性约束：
- 不得直接使用 Ant Design Vue 默认字号、颜色、图标尺寸和连接线样式。
- 步骤图标、标题、描述、连接线必须按本规范统一覆盖。
- 颜色必须优先使用现有设计 token，不允许新增未定义色值。

---

## 7. 验收标准

1. 横向步骤条和纵向步骤条结构清晰。
2. 图标、标题、描述之间的间距稳定。
3. 四种步骤状态（进行中/待进行/已完成/错误）视觉区分明确。
4. 连接线样式为虚线，颜色随步骤状态正确切换。
5. 步骤图标尺寸统一为 24px × 24px。
6. 进行中步骤图标为蓝色实心圆 + 白色序号。
7. 已完成步骤图标为蓝色描边圆 + 蓝色对勾。
8. 待进行步骤图标为灰色描边圆 + 灰色序号。
9. 错误步骤图标为红色描边圆 + 红色感叹号。
10. 步骤数量在 2-7 之间，超出时已拆分流程。
11. 纵向步骤条右侧内容区按钮位置正确，不在步骤条内部。
12. 使用 Ant Design Vue Steps 底座，未自行造轮子。
13. 所有颜色使用设计 token，无未定义色值。
