# 单选框组件 (Radio)

## ⚠️ 高频违规速查（阅读正文前必看）

> AI 在生成单选框代码前，必须先逐行核对此表，再阅读下方完整规范。

| 规则 | ✅ 正确 | ❌ 常见错误 |
|------|--------|------------|
| 单选框尺寸 | 固定 `16px × 16px` | 使用其他尺寸 |
| 未选中线圈颜色 | `var(--color-graphite-l20)` | 使用深色或黑色边框 |
| 已选中线圈颜色 | `var(--color-blue)` | 使用其他颜色 |
| 已选中中心圆点 | `8px` 实心圆，颜色 `var(--color-blue)` | 圆点过大或使用其他颜色 |
| 文字字号 | `12px`，字重 `400` | 使用 `14px` 或加粗 |
| 文字与按钮间距 | `4px` | 使用 `8px` 或更大间距 |
| 关联配置面板对齐 | 面板左边缘与**表单控件区域**左边缘对齐 | 从页面最左侧、Label 左边缘或弹窗边缘开始 |
| 关联配置面板箭头 | 必须有**顶部小三角**，指向当前选中 Radio 选项 | 无箭头，或箭头固定在面板左上角不跟随选中项 |
| 关联配置面板背景 | `var(--color-graphite-l50)`，边框 `1px var(--color-graphite-l30)` | 使用白色或无边框 |
| 关联配置面板内边距 | 上下 `8px`，左右 `16px` | 使用 AntD Card 默认 padding |
| 悬浮态线圈颜色 | 变为 `var(--color-blue)` | 无悬浮态反馈 |

## 使用声明

本规范只定义组件的视觉结构、布局关系、交互行为和验收标准，不提供具体框架代码实现。

生成代码时，必须以当前项目的技术栈 Skill 为准。

如果本规范与项目技术栈约束发生冲突，优先遵守项目技术栈约束。

本规范禁止包含：
- React / Vue / Angular 代码示例
- JSX / TSX
- TypeScript 示例
- import 语句
- className
- React state 写法
- 具体图标组件名
- 具体组件库 API

本规范中的组件名称仅表示设计语义，不代表必须使用某个框架或组件库的同名组件。

本组件遵循 [全局原子规范与间距规范](design-atomic-spacing.md) 中的设计原则。

## 基础单选 - 子弹头样式

### 未选状态 (Unchecked)
- 尺寸：固定 16px x 16px
- 线圈：1px
- 线圈颜色：var(--color-graphite-l20)
- 中心填充：透明

### 已选状态 (Checked)
- 尺寸：固定 16px x 16px
- 线圈：1px
- 线圈颜色：var(--color-blue)
- 中心填充：8px 实心圆点，颜色 var(--color-blue)

### 未选禁用状态 (Unchecked Disabled)
- 线圈颜色：var(--color-graphite-l20)
- 中心填充色：var(--color-graphite-l30)

### 已选禁用状态 (Checked Disabled)
- 线圈颜色：var(--color-graphite-l20)
- 中心填充色：var(--color-graphite-l30)
- 中心圆点颜色：var(--color-graphite)

### 单选按钮带文字
- 文字与按钮间距：4px
- 文字字号：12px
- 字重：400
- 文字颜色：var(--color-graphite-d40)
- 禁用文字颜色：var(--color-graphite)

## 交互态规范

### 悬浮态 (Hover)
- 未选中悬浮：线圈颜色变为 var(--color-blue)
- 已选中悬浮：线圈颜色保持 var(--color-blue)

### 点击态 (Active)
- 未选中点击：线圈颜色变深 var(--color-blue-d10)
- 已选中点击：线圈颜色变深 var(--color-blue-d10)

## 单选项关联配置面板规范

### 组件定位

当表单中的某个 Radio 单选项被选中后，如果该选项需要填写额外配置，应在该单选项下方展示"关联配置面板"。

关联配置面板属于当前单选项的从属内容，不是独立 Card、Collapse、Drawer 或 Modal。

典型场景：

```
存储空间
├─ 内置
└─ 外部服务器  ← 选中
   └─ 外部服务器配置面板
      ├─ 协议
      ├─ 服务器地址
      ├─ 端口
      ├─ 账号
      ├─ 密码
      └─ 存储路径
```

### 显示规则

- 仅当对应 Radio 选项被选中时显示关联配置面板
- 未选中时不显示该面板
- 面板必须紧跟 Radio 选项所在行下方
- 面板内容属于当前 Radio 选项，不得与其他表单区块混淆
- 面板切换显示/隐藏时，不得造成主表单整体布局错位

### 面板位置

- 面板左侧应与表单控件区域左侧对齐
- 不应从弹窗最左侧开始铺满
- 不应与表单 Label 区域对齐
- 面板宽度应占满当前表单控件区域的剩余宽度
- 面板顶部与 Radio 行之间保持紧凑间距

错误示例：

```
存储空间  ○ 内置  ● 外部服务器
[面板从页面最左侧开始]   ❌
```

正确示例：

```
存储空间  ○ 内置  ● 外部服务器
          [外部服务器配置面板]   ✅
```

### 面板箭头

关联配置面板顶部必须有一个小箭头，指向当前选中的 Radio 选项。

- 箭头位于面板顶部边框上
- 箭头方向朝上
- 箭头应指向被选中的 Radio 选项区域
- 箭头颜色应与面板背景一致
- 箭头边框颜色应与面板边框一致
- 箭头不得偏离选中项，也不得固定在面板左上角

当选中项为"外部服务器"时，箭头应指向"外部服务器"单选项。

### 面板样式

| 属性 | 规范 |
|------|------|
| 背景色 | 使用一级背景色 var(--color-graphite-l50) |
| 边框 | 1px 一级分割线颜色 var(--color-graphite-l30) |
| 圆角 | 2px |
| 内边距 | 上下 8px，左右 16px |
| 箭头 | 顶部小三角，指向选中 Radio |
| 阴影 | 无阴影 |
| 展开方式 | 直接展开，不使用浮层 |

面板内边距固定为：上下 8px，左右 16px。

不得使用 Ant Design Vue Card 默认 padding。

### 面板内表单规则

关联配置面板内部仍然属于表单区域，因此必须继续遵循项目已有表单规范。

面板内表单项应继承：

- 表单 Label 字号、颜色、宽度
- 必填星号样式
- 输入框高度
- Select 高度
- Input Password 样式
- Placeholder 样式
- Info 提示图标样式
- 表单项上下间距
- 校验错误样式

不得因为放入面板而重新定义另一套表单样式。

### 面板内提示信息

如果面板顶部有说明文案，应使用表单提示条样式。

示例：

> ⓘ 系统将在每次自动备份完成后，将生成的备份文件实时上传至指定服务器

要求：

- 提示信息位于面板内容顶部
- 提示条宽度占满面板内容区
- 提示条与下方表单项保持固定间距
- 不得使用强警告色
- 不得使用弹窗提示或 Tooltip 替代

## 验收标准
- 单选框尺寸固定 16px x 16px
- 已选状态中心圆点清晰可见
- 禁用状态视觉可区分
- 悬浮和点击态颜色反馈正确
- 单选框组内互斥，同时只能选中一个

### 关联配置面板验收标准

- 面板仅在对应 Radio 选中时显示，未选中时隐藏
- 面板左侧与表单控件区域左边缘对齐
- 面板顶部箭头指向当前选中的 Radio 选项
- 面板背景色为一级背景色，边框为一级分割线颜色
- 面板内边距为上下 8px、左右 12px
- 面板内表单项继承主表单规范，无样式差异
- 面板切换时不造成主表单布局错位
