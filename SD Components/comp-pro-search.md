# 复合搜索框 (ProSearch) 规范

> 本文档中"复合搜索框"与"高级搜索框"为同一组件的不同叫法，以下统称"复合搜索框"。
>
> 收藏条件功能（第 11 章）默认不包含，仅在业务需求明确要求支持收藏条件、或要求给复合搜索框增加收藏条件组件时，才实现该功能。

## 1. 组件结构

复合搜索框由输入框和搜索按钮合并为一个整体控件。

结构示意：

[输入框区域][搜索按钮]

合并规则：

- 输入框和搜索按钮合并在一起，中间没有圆角，只保留 1px 分割线
- 合并后的整体控件四周圆角为 2px
- 不得出现双边框，不得出现各自独立的圆角

## 2. 物理尺寸与自适应逻辑

### 2.1 容器尺寸

| 规范项 | 规范值 |
|--------|--------|
| 默认宽度 | min(660px, 100%) |
| 最小宽度 | min(480px, 100%) |
| 最大宽度 | 100%（父容器宽度） |
| 单行高度 | 32px |
| 整体圆角 | 2px |
| 边框颜色 | var(--color-graphite-l20) |
| 背景色 | var(--color-white) |
| 默认提示语 | 点击选择筛选条件或直接输入搜索内容，按回车键进行检索 |

高度规则：

- 默认高度为 32px
- 当 Tag 过多需要换行时，容器高度允许随内容增长
- 单行最小高度必须保持 32px
- 搜索按钮区域高度固定 32px，并始终贴齐容器顶部；当容器因 Tag 换行而增高时，搜索按钮仍保持 32px 高度，不随容器高度拉伸

宽度约束：

- 当父容器宽度小于 480px 时，允许宽度压缩到父容器宽度，不得撑破父容器
- 严禁在输入过程中发生宽度抖动或跳变
- 禁止宽度动画过渡

### 2.2 容器融合

- 复合搜索框必须是一个高度整合的输入容器
- 所有元素（Tags、输入文本、清空图标、搜索按钮）必须包含在同一个外边框内

## 5. 交互逻辑

### 5.1 面板唤醒与实时性

- 当用户在输入框内键入任何字符时，必须立即弹出快捷面板
- 面板 open 状态必须在 focus/input 事件同步更新，不得被 debounce、setTimeout 或异步请求阻塞；若需要使用 nextTick 或 requestAnimationFrame 计算面板位置，不得影响面板的立即显示
- 搜索候选数据可以 debounce 100-200ms，但面板容器必须立即显示，可先展示 loading、空列表或上一次结果
- 用户键入第一个字符的瞬间，面板必须完成从"空状态"到"输入态"的切换

### 5.2 即时联动

- 面板中的搜索字样必须随用户的每一次按键实时更新

### 5.3 默认搜索逻辑

- 若用户未在面板中手动选择字段，直接按回车或点击搜索按钮时，默认使用 `default: true` 的字段执行搜索
- 若没有字段标记 `default: true`，则使用 `fields[0]`

### 5.4 双态面板

#### 空状态面板

- 触发条件：Focus 但未输入内容
- 展示内容：当前支持搜索的字段清单，垂直列表布局
- 严禁使用横向平铺的标签样式

#### 输入态面板

- 触发条件：开始输入字符
- 展示内容：动态模糊匹配选项
- 格式：以 "[字段名]：[输入内容]" 搜索
- 垂直列表布局

### 5.5 键盘交互

- **Enter**：
  - 输入态：若存在高亮候选项，使用高亮候选项生成 Tag；若不存在高亮候选项但存在当前输入文本，则使用默认字段将当前输入文本生成搜索条件并触发搜索；若当前输入文本为空，则不生成新条件，仅触发已有条件搜索
  - 空状态：若有高亮字段，执行该字段点击逻辑
- **Backspace**：输入为空且无编辑态 Tag 时，删除最后一个 Tag；输入为空且存在编辑态 Tag 时，移除该编辑态 Tag
- **Esc**：关闭面板，但不清空已输入内容
- **ArrowUp / ArrowDown**：在当前面板列表中移动高亮项

## 7. 字段联动与编辑态 Tag 规则

字段列表中的字段项不是普通展示文本，而是创建搜索条件 Tag 的入口。用户点击字段后，组件必须立即在搜索框内生成一个对应字段的编辑态 Tag。输入型字段和选择型字段都必须先生成 Tag，再继续完成值输入或值选择。

### 7.1 Tag 状态定义

搜索条件 Tag 分为三种状态：

| 状态 | 说明 |
|------|------|
| 编辑态 Tag | 字段已确定，但值尚未完成，用户正在输入或选择 |
| 完成态 Tag | 字段和值都已确定，可参与搜索 |
| 无效 Tag | 字段存在但值为空，不得参与搜索 |

规则：

- 点击字段后必须立即生成编辑态 Tag
- 编辑态 Tag 显示在搜索框内，而不是只显示普通文本前缀
- 编辑态 Tag 内部必须支持输入、删除、关闭
- 完成态 Tag 才能作为有效搜索条件参与搜索
- 无效 Tag 不得参与搜索、收藏或提交

### 7.2 输入型字段联动规则

适用字段：设备名称、源 IP、目的 IP、IP / 终端名、责任人、应用、服务、协议名称及其他需要自由输入的字段。

点击输入型字段后，必须执行：

1. 关闭字段清单面板
2. 立即在搜索框内生成一个编辑态 Tag
3. Tag 内展示字段名、分隔符和值输入区域
4. 光标自动定位到该 Tag 的值输入区域
5. 用户直接在 Tag 内输入完整值
6. 输入完成后，该 Tag 转为完成态 Tag

编辑态 Tag 示例：`IP / 终端名：|请输入 ×`

用户输入后：`IP / 终端名：10.10.10.1,196.168.10.1 ×`

完成态 Tag 示例：`IP / 终端名：10.10.10.1,196.168.10.1 ×`

规则：

- 输入型字段点击后，不得只生成普通文本前缀
- 输入型字段点击后，不得等用户按 Enter 才生成 Tag
- 用户输入内容必须发生在 Tag 内部
- Tag 内的值输入区域必须支持光标、输入、删除和清空
- 输入值为空时，该 Tag 为编辑态或无效态，不得参与搜索
- 用户按 Enter、点击搜索按钮、点击其他区域或选择其他字段时，可将当前编辑态 Tag 确认为完成态 Tag
- 若编辑态 Tag 没有输入值，按 Esc、点击关闭 X 或失焦时可移除该 Tag

禁止：

- 禁止点击字段后只把字段名作为普通文字插入输入框
- 禁止用户输入值出现在 Tag 外部
- 禁止生成"字段名：值：值"的重复内容
- 禁止空值 Tag 参与搜索

### 7.3 选择型字段联动规则

适用字段：动作、风险等级、处理状态及其他固定枚举值字段。

点击选择型字段后，必须执行：

1. 关闭字段清单面板
2. 立即在搜索框内生成一个编辑态 Tag
3. Tag 内展示字段名和值占位内容
4. 同时打开该字段对应的值选择悬浮面板
5. 用户在悬浮面板中选择具体值
6. 点击确定后，将选择值写入该 Tag
7. 该 Tag 转为完成态 Tag

编辑态 Tag 示例：`动作：请选择 ×`

选择值后：`动作：允许、阻断 ×`

规则：

- 选择型字段点击后必须先生成 Tag
- 选择型字段点击后必须继续打开值选择面板
- 值选择面板用于补全当前编辑态 Tag 的值
- 用户点击确定后，必须更新当前 Tag，而不是新建另一个 Tag
- 用户点击取消时，若该 Tag 还没有有效值，应移除该编辑态 Tag
- 若该 Tag 原本已有值，取消时应恢复原值
- 未确认的临时选择不得参与搜索
- 未确认的临时选择不得保存到收藏条件

选择面板结构：

- 全选复选框
- 各选项复选框
- 底部：取消按钮 + 确定按钮，左对齐，按钮间距 8px
- 底部按钮区域有顶部分割线
- 按钮最小宽度 56px，高度 28px

禁止：

- 禁止点击选择型字段后只打开面板、不生成 Tag
- 禁止选择值后重新生成一个重复 Tag
- 禁止取消选择后留下空值 Tag

### 7.4 自由输入候选项生成 Tag 规则

当用户未先选择字段，而是直接在搜索框中输入内容时，组件进入自由输入态。

用户输入内容后，候选面板展示：

- 以"关键字：输入内容"搜索
- 以"设备名称：输入内容"搜索
- 以"设备SN码：输入内容"搜索
- 以其他字段搜索

用户点击候选项或按 Enter 确认候选项后，必须立即生成完成态 Tag。

示例：用户直接输入 `124563434`，点击候选项"以'设备名称：124563434'搜索"后，生成 Tag：`设备名称：124563434 ×`

规则：

- 直接输入内容时，输入值临时存在于主输入区域
- 一旦用户选择候选项或按 Enter 确认，必须把输入值转换为完成态 Tag
- 转换完成后，主输入区域中的原始输入值必须清空
- 同一次输入只能生成一个 Tag
- 不得同时生成自由输入 Tag 和默认字段 Tag
- 不得出现重复值

禁止：

- 禁止生成"设备名称：124563434：124563434"
- 禁止生成 Tag 后输入框中仍残留原始输入值
- 禁止同一次 Enter 触发两次 Tag 创建

### 7.5 Enter 键规则

Enter 的行为必须根据当前状态区分：

**当前存在输入型编辑态 Tag**：若 Tag 内已有输入值，按 Enter 将该 Tag 确认为完成态 Tag；若 Tag 内无输入值，按 Enter 不得生成有效搜索条件；不得新建额外 Tag。

**当前存在选择型编辑态 Tag**：若值选择面板已打开，Enter 可等价于点击确定；若未选择任何值，Enter 不得生成有效搜索条件；不得新建额外 Tag。

**当前为自由输入态**：若候选面板存在高亮项，按 Enter 使用高亮候选项生成完成态 Tag；若没有高亮项，按 Enter 使用第一项默认候选生成完成态 Tag；生成 Tag 后必须清空主输入内容。

**当前没有输入内容，也没有编辑态 Tag**：按 Enter 触发已有完成态条件的搜索。

### 7.6 编辑态 Tag 内部结构

编辑态 Tag 必须支持内部输入或内部选择。

输入型编辑态 Tag 结构：

`[字段名][分隔符][值输入区域][关闭 X]`

- 字段名：不可编辑，使用当前字段 label
- 分隔符：中文冒号"："或英文冒号":"，项目统一
- 值输入区域：可编辑，支持光标定位、输入、删除、清空
- 关闭 X：点击移除整个 Tag

选择型编辑态 Tag 结构：

`[字段名][分隔符][值占位文本][关闭 X]`

- 字段名：不可编辑
- 分隔符：与输入型一致
- 值占位文本：初始为"请选择"，选择后替换为实际值
- 关闭 X：点击移除整个 Tag

完成态 Tag 结构：

`[字段名][分隔符][值文本][关闭 X]`

- 字段名、分隔符、值文本均不可编辑
- 关闭 X：点击移除整个 Tag

### 7.7 Tag 视觉区分

| 状态 | 背景色 | 边框色 | 说明 |
|------|--------|--------|------|
| 编辑态 | var(--color-graphite-l50) | var(--color-blue) | 蓝色边框标识正在编辑 |
| 完成态 | var(--color-graphite-l50) | var(--color-graphite-l30) | 灰色边框标识已完成 |
| 无效态 | var(--color-graphite-l50) | var(--color-graphite-l30) | 视觉同完成态，但值为空 |

编辑态 Tag 与完成态 Tag 必须通过边框颜色明确区分。

## 8. 焦点保持与稳定性

### 8.1 焦点保持

- 在输入过程中，严禁重新渲染外部容器
- 输入框实体必须在整个生命周期内保持稳定
- 仅允许更新内部的文本值和面板内容
- 禁止因为状态改变而导致输入焦点丢失

### 8.2 激活逻辑

| 规范项 | 规范值 |
|--------|--------|
| 单次点击激活 | 点击搜索框容器任何区域，必须立即触发 Focus 并唤醒面板 |
| 禁止双击 | 严禁要求双击或多次点击才能激活 |
| 防抖优化 | 仅对搜索匹配逻辑增加 100-200ms 防抖，面板显示/隐藏不得防抖 |

### 8.3 空间溢出保护

| 规范项 | 规范值 |
|--------|--------|
| 容器溢出 | 外层容器严禁设置 overflow: hidden |
| 下拉面板定位 | 必须使用绝对定位，确保能跳出父容器边界 |
| 面板层级 | z-index 必须足够高，不被父容器截断 |
| Teleport | 若必须使用 overflow: hidden 实现圆角，面板必须通过 Teleport 挂载到 body |

### 8.4 稳定性约束

| 类别 | 必须 | 禁止 |
|------|------|------|
| 面板显示 | 同步即时响应 | 对显示/隐藏逻辑防抖 |
| 状态切换 | 同步执行 | 延迟包裹 |
| 容器溢出 | overflow: visible | overflow: hidden |
| 面板层级 | 高 z-index 或 Teleport | 被父容器截断 |

### 8.5 激活态视觉反馈

| 规范项 | 规范值 |
|--------|--------|
| 边框颜色 | var(--color-blue) (#1C6EFF) |
| 投影效果 | 0 4px 16px rgba(30, 35, 43, 0.14) |
| 禁止 | 禁止 box-shadow 出现蓝色光晕 |

## 9. 实现约束

当项目技术栈为 Vue 3 + Ant Design Vue 时：

- 必须封装为独立组件 `ProSearch.vue`
- 不得直接使用 `a-select`、`a-auto-complete` 或 `a-input-search` 作为主容器实现
- 主输入区域必须使用稳定存在的原生 `input` 元素
- Ant Design Vue 只可用于 Button、Checkbox、Icon、Tooltip 等基础展示或交互零件，不得用 Select、AutoComplete、Input.Search 替代主容器
- 下拉面板必须通过 Vue Teleport 挂载到 body，避免被父容器 overflow 截断
- 面板显示状态必须同步更新，搜索匹配结果可以 debounce

## 10. 数据契约

### 10.1 字段配置契约

以下结构仅定义字段配置的数据含义，不限制具体代码写法、类型系统或文件组织方式。

字段配置必须包含：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| key | string | 是 | 字段唯一标识 |
| label | string | 是 | 面板和 Tag 中展示的字段名称 |
| type | input / select | 是 | input 表示自由输入，select 表示固定选项 |
| default | boolean | 否 | 是否作为默认搜索字段 |
| options | Array | 否 | 选择型字段的可选项，仅 type 为 select 时使用 |

选择项结构：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| label | string | 是 | 选项展示名称 |
| value | string / number | 是 | 选项实际值 |

规则：

- `type: input` 为输入型字段
- `type: select` 为选择型字段
- 若没有字段配置标记 `default: true`，则默认使用字段列表中的第一个字段作为默认搜索字段

### 10.2 搜索输出契约

以下结构仅定义搜索事件输出的数据含义，不限制具体代码写法、类型系统或文件组织方式。

搜索条件必须包含：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| fieldKey | string | 是 | 字段唯一标识 |
| fieldLabel | string | 是 | 字段展示名称 |
| operator | eq / contains / in | 否 | 查询操作符 |
| value | string / number / array | 是 | 查询值 |
| label | string | 是 | Tag 中展示的完整文本 |

搜索事件 payload 必须包含：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| conditions | Array | 是 | 已生成的搜索条件列表 |
| keyword | string | 否 | 当前未转换为 Tag 的输入文本 |
| rawInput | string | 否 | 原始输入文本 |

规则：

- 点击搜索按钮或按 Enter 时，组件必须触发 `search` 事件
- `search` 事件必须输出上述 payload
- 若当前输入内容已经被转换为 Tag，则放入 `conditions`
- 若当前输入内容尚未转换为 Tag，则可放入 `keyword` 或根据默认字段转为条件

## 11. 搜索核心区与操作按钮区边界

高级搜索区域由两个独立部分组成：

[搜索核心区][收藏条件按钮组]

### 11.1 搜索核心区

搜索核心区包含：

- 输入区域
- Tag 展示区域
- 清空图标
- 搜索按钮

搜索核心区负责输入、筛选条件展示、字段面板联动和搜索提交。

激活态边框、激活态阴影、输入焦点状态只能作用于搜索核心区，不得作用到收藏条件按钮组。

### 11.2 收藏条件按钮组

收藏条件按钮组包含：

- 收藏 icon 按钮
- 下拉 icon 按钮

收藏条件按钮组是独立操作区，不属于搜索核心区的输入外框。

收藏条件按钮组与搜索核心区之间必须保留 8px 间距。

### 11.3 激活态边界

当搜索框 Focus 或输入时：

- 只允许搜索核心区出现激活态边框
- 收藏条件按钮组不得被蓝色边框包住
- 收藏条件按钮组不得跟随搜索核心区出现激活态阴影
- 收藏条件按钮组只能在自身 hover、active 或面板展开时改变自身状态

禁止：

- 禁止将搜索框、搜索按钮、收藏按钮组包在同一个激活态边框内
- 禁止蓝色边框从输入区域延伸到收藏按钮组
- 禁止收藏按钮组参与输入框 Focus 样式

## 12. 收藏条件功能

收藏条件功能用于将当前复合搜索框中的筛选条件保存为可复用的快捷筛选项。该功能由收藏按钮与下拉按钮组成，位于搜索按钮右侧。

### 12.1 组件结构

收藏条件功能由两个按钮拼接组成一个完整按钮组：

```
[搜索按钮 32px] 8px间距 [收藏按钮 32px][下拉按钮 32px]
```

结构规则：

- 收藏按钮与下拉按钮必须拼接为一个整体按钮组
- 两个按钮之间只允许出现 1px 分割线
- 拼接位置不得出现圆角
- 按钮组整体四周圆角为 2px
- 不得出现双边框
- 不得出现两个按钮各自独立的圆角
- 收藏按钮组与左侧蓝色搜索按钮之间的间距固定为 8px
- 禁止把收藏按钮放入搜索框输入区域内部

### 12.2 按钮尺寸与样式

| 规范项 | 规范值 |
|--------|--------|
| 按钮组高度 | 32px |
| 收藏按钮宽度 | 32px |
| 下拉按钮宽度 | 24px |
| 按钮组圆角 | 2px |
| 边框颜色 | var(--color-graphite-l20) |
| 背景色 | var(--color-white) |
| 分割线颜色 | var(--color-graphite-l20) |
| 与搜索按钮间距 | 8px |

收藏按钮样式：

| 状态 | 样式 |
|------|------|
| 默认状态 | 空心星形图标，颜色 var(--color-graphite-d10) |
| Hover 状态 | 图标颜色变为 var(--color-blue) (#1C6EFF) |
| 激活状态 | 边框颜色为 var(--color-blue) (#1C6EFF) |
| 已收藏状态 | 星形图标可使用蓝色填充或蓝色高亮 |
| 禁用状态 | 图标颜色 var(--color-graphite-l10)，鼠标不可点击 |

下拉按钮样式：

| 状态 | 样式 |
|------|------|
| 默认状态 | 向下箭头图标，颜色 var(--color-graphite-d10) |
| Hover 状态 | 图标颜色变为 var(--color-blue) (#1C6EFF) |
| 面板展开状态 | 箭头可切换为向上箭头，边框颜色为 var(--color-blue) (#1C6EFF) |
| 禁用状态 | 图标颜色 var(--color-graphite-l10)，鼠标不可点击 |

### 12.3 启用与禁用规则

收藏按钮：

- 当前搜索框存在至少一个有效搜索条件时，收藏按钮可点击
- 当前搜索框不存在任何有效搜索条件时，收藏按钮禁用
- 无效条件包括：空输入、未完成的编辑态 Tag、未确认的选择型字段临时选择

下拉按钮：

- 下拉按钮用于查看已收藏的搜索条件
- 下拉按钮不依赖当前搜索框是否存在条件
- 若暂无收藏数据，点击后展示空状态面板
- 若存在收藏数据，点击后展示收藏列表面板

### 12.4 收藏保存面板

点击收藏按钮后，展示收藏保存面板。

触发条件：当前搜索框中存在有效搜索条件，用户点击收藏按钮。

面板内容：

- 收藏名称输入项
- 确定按钮
- 取消按钮

面板结构：

```
收藏名称  [请输入收藏名称]
[确定] [取消]
```

样式规范：

| 规范项 | 规范值 |
|--------|--------|
| 面板宽度 | 256px |
| 面板背景 | var(--color-white) |
| 面板圆角 | 2px |
| 面板阴影 | var(--seer-dropshadow-s1) |
| 面板与按钮间距 | 4px |
| 面板内边距 | 16px |
| 标签文字颜色 | var(--color-graphite-d20) |
| 输入框高度 | 32px |
| 按钮高度 | 24px |
| 按钮对齐 | 右对齐，确定在左，取消在右 |
| 按钮间距 | 8px |

交互规则：

- 打开收藏保存面板后，收藏名称输入框自动聚焦
- 用户点击确定后，校验收藏名称
- 收藏名称不能为空
- 校验通过后，将当前搜索条件保存为一条收藏记录
- 保存成功后，关闭收藏保存面板
- 保存成功后，该收藏记录必须出现在下拉按钮的收藏列表面板中
- 点击取消、面板外部或按 Esc 时，关闭面板，不保存数据

收藏名称校验：

- 收藏名称不能为空
- 收藏名称前后空格必须自动去除
- 若存在同名收藏，必须给出明确处理方式：覆盖原收藏、提示重名或禁止保存
- 默认建议：同名时禁止保存，并提示用户修改名称

### 12.5 收藏列表面板

点击下拉按钮后，展示收藏列表面板。

面板内容：

- 操作区（清空、刷新）
- 收藏搜索输入框
- 收藏列表（表格形式）
- 已添加数量

面板结构：

```

─────────────────────────────────────────
序号 | 收藏名称 | 筛选项 | 操作
```

| 面板圆角 | 2px |

表格相关样式（表头背景、行高、分割线等）遵循 comp-table.md 表格组件规范。

可点击列：

- 收藏名称：文字颜色 var(--color-blue)，可点击应用该收藏
- 操作列（删除）：文字颜色 var(--color-blue)，可点击删除

其他文字：

- 筛选项摘要：var(--color-graphite-d20)，12px
- 底部已添加数量：var(--color-graphite-d10)，12px

交互规则：

- 点击收藏名称行：将该收藏的搜索条件应用到搜索框中
- 应用收藏后，搜索框中生成对应的 Tag，并自动触发搜索
- 点击删除按钮：删除该条收藏记录
- 点击清空按钮：删除所有收藏记录
- 点击刷新按钮：重新加载收藏列表
- 收藏列表支持滚动，最大高度建议 320px
- 收藏数量达到上限时，收藏按钮禁用或给出提示

## 13. 验收标准

### 13.1 视觉验收

- 初始单行高度为 32px
- 默认宽度为 660px；父容器不足时不得溢出父容器
- 搜索按钮宽高均为 32px，位于最右侧
- 输入区域与搜索按钮共用一个外边框
- 搜索按钮不得出现独立圆角
- Tag 背景为 var(--color-graphite-l40)，边框为 var(--color-graphite-l30)，文字为 var(--color-graphite-d40)
- Tag 关闭图标为灰色 X，不得使用蓝色圆形背景

### 13.2 交互验收

- 单击容器任意位置后，input 必须获得焦点
- Focus 且无输入时，立即显示字段列表面板
- 输入第一个字符时，面板必须立即切换为输入态
- 搜索匹配可以 debounce 100-200ms，但面板 open 状态不得 debounce
- 按 Enter 或点击搜索按钮必须触发 search 事件，payload 使用搜索输出契约结构
- 未选择字段时，默认使用 default: true 的字段或 fields[0] 生成搜索条件
- 点击选择型字段后，必须切换为多选面板
- 点击输入型字段后，必须生成编辑态 Tag
- Backspace 可删除最后一个 Tag 或取消编辑态 Tag
- Esc 关闭面板但不清空已输入内容
- ArrowUp / ArrowDown 在面板列表中移动高亮项

### 13.3 稳定性验收

- 输入过程中 input DOM 不得被销毁重建
- 输入过程中不得丢失焦点
- 输入过程中容器宽度不得变化
- 父容器存在 overflow hidden 时，面板仍不得被截断
