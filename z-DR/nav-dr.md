# DR 导航框架规范

## 使用声明

本规范只定义 DR 导航的视觉、结构、交互和数据规则，不提供任何具体代码实现。

生成代码时，必须以当前项目的技术栈 Skill 为准。

如果本规范中的设计描述与项目技术栈约束发生冲突，优先遵守项目技术栈约束。

本规范禁止包含：
- React / Vue / Angular 代码示例
- JSX / TSX / TypeScript 示例
- import 语句
- 具体组件库 API 示例
- 具体图标组件名称
- 具体框架状态管理代码

本规范中的组件名称如 Header、Sider、Tabs、Select、Menu 仅表示设计语义，不代表必须使用某个框架或组件库的同名组件。

| 层级 | 视觉标识 | 左侧坐标 | 间距说明 | 字号 |

## 4. 视觉状态红线 (Visual States)

### 4.1 选中态 (Selected)

| 属性 | 规范值 |
|-----|--------|
| 背景 | 蓝色渐变，从左侧 rgba(28,110,255,0.2) 渐变至右侧透明 |
| 左侧激活条 | 3px 垂直蓝色条，紧贴左边缘，不影响图标和文字的原始位置 |
| 文字颜色 | 纯白 #FFFFFF |
| 图标位置 | 不变（选中与未选中图标和文字位置完全一致） |
| 一级菜单字重 | 600（未选中为 400） |
| 二级菜单字重 | 600（未选中为 400） |

**适用范围**：
- 二级菜单可进入选中态
- 一级菜单如果没有包含子菜单（即本身为叶子节点），也可以进入选中态，此时图标变蓝（#1C6EFF）
- 一级菜单如果包含子菜单，则不进入选中态，而是触发父级高亮（见 4.2）

**重要约束**：
- 选中的蓝色条不能影响菜单图标和文字的位置
- 选中和未选中时候图标和文字的位置是不变的，不要产生右移

### 4.2 父级高亮 (Parent Active)

当一级菜单包含二级菜单且子菜单选中时，触发以下效果：

| 属性 | 规范值 |
|-----|--------|
| 一级菜单图标颜色 | #1C6EFF (blue) |
| 一级菜单文字颜色 | 保持原色（白色），不变色 |
| 一级菜单底色 | #1E232B |
| 同层级二级菜单底色 | #1E232B |

**重要约束**：
- 二级菜单被选中时，该二级菜单所在的一级菜单以及同一层级的二级菜单的底色改为 #1E232B
- 一级菜单仅图标变色（blue），文字保持白色不变

## 5. 交互与联动逻辑

### 5.1 自动展开与互斥收起

| 规则 | 说明 |
|------|------|
| 默认展开 | 选中二级菜单时，该二级菜单所在的一级菜单必须默认展开，不能收起 |
| 保证可见 | 确保选中的菜单能够直接被看到 |
| 互斥收起 | 同时只允许一个一级菜单展开，点击新的菜单时自动收起其他已展开的一级菜单 |

**实现逻辑**：
1. 维护当前展开的一级菜单 Key 列表
2. 点击新的菜单时，自动移除其他已展开的菜单 Key
3. 只保留当前点击的菜单在展开状态

### 5.2 折叠交互

| 规则 | 说明 |
|------|------|
| 收起态宽度 | 54px |
| Hover 交互 | 鼠标 Hover 区域需临时悬浮展开完整菜单，浮层带阴影效果 |
| 恢复机制 | 移开后恢复 54px 收起态 |
| 收起态隐藏 | 文字、产品名称、用户名、展开箭头全部隐藏 |
| 收起态图标 | 一级菜单图标水平居中，左边距 19px（(54-16)/2） |
| 收起态用户区 | 头像居中显示，取消左间距 |

## 6. 样式强制装配要点

以下为必须强制实现的视觉规范，无论使用何种技术栈均需确保：

1. **一级菜单图标位置**：左边距 16px，尺寸 16px × 16px，与文字间距 8px
2. **二级菜单文字起始位置**：左边距 40px
3. **选中态蓝色激活条**：3px 垂直蓝色边框 + 蓝色渐变底色（从左到右，20%蓝色透明度到透明），必须不影响图标和文字的原始位置
4. **父级高亮**：子菜单选中时，一级菜单图标变蓝（#1C6EFF），文字不变色，同层级底色 #1E232B
5. **叶子节点选中**：无子菜单的一级菜单选中时，图标变蓝（#1C6EFF）
6. **一级菜单字号**：13px，选中字重 600，未选中字重 400
7. **二级菜单字号**：12px，选中字重 600，未选中字重 400
8. **互斥展开**：同时只允许一个一级菜单展开
9. **收起态**：54px 宽度，图标居中（左边距 19px），文字和展开箭头隐藏

## 8. 设计原则

| 原则 | 说明 |
|------|------|
| 深色一致性 | 侧边栏整体保持 #272E38 深色背景，所有状态变化在深色基础上叠加 |
| 坐标严格性 | 一级菜单图标 16px 左边距、二级菜单 40px 起始坐标必须严格执行 |
| 布局稳定性 | 选中态蓝色条不影响图标和文字的原始位置 |
| 互斥展开 | 同一时间只允许一个一级菜单展开，避免导航过长 |
| 父级联动 | 子菜单选中时父级图标变蓝 + 同层级底色变化，形成视觉联动 |

## 9. 注意事项

- 选中态蓝色条不能改变图标和文字的水平位置
- 一级菜单无子菜单时可直接进入选中态（图标变蓝 + 文字变白）
- 父级高亮仅图标变蓝，文字保持原色
- 同层级底色 #1E232B 仅在子菜单选中时触发
- 收起态 Hover 展开后移开鼠标自动恢复 54px
- 选中二级菜单时必须自动展开其父级菜单
- 禁止选中态影响图标和文字的水平位置
- 禁止同时展开多个一级菜单
- 禁止父级高亮时文字变色
